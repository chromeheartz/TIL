### 📌 웹의 작동 방식

![](https://velog.velcdn.com/images/chromeheartz/post/d942e713-c758-47cb-bba4-046ef8a718a9/image.png)

웹의 작동하는 방식은 `유저 / 클라이언트` 가 있고, 웹 페이지를 방문하거나 이미 웹 페이지에 접속한 상태로 양식을 보내는 등의 `상호작용` 이 있다.
홈페이지를 방문할 때 브라우저에 `URL` 을 입력하면 실제로 뒤에서 일어나는 일은 브라우저가 도메인 이름 서버로 접근하여 해당 도메인을 찾게 된다.

도메인을 찾는 이유는이 도메인이 서버의 실제 주소에 해당하지 않기 때문이다.
쉽게 말하면 해당 주소를 사람이 읽을 수 있도록 `인코딩` 한 버전인것이다.
서버의 경우 단순히 IP주소를 가지고 있으나 이건 화면 뒤에서 일어나는 기술적 요소로 최종적으로 URL을 입력하게 되면 `어딘가의 서버` 로 연결된다.

따라서 브라우저는 언급했던 주어진 IP주소를 사용하여 해당 서버로 요청을 보내게 되며 그 IP주소는 `도메인` 에 속한다.

해당 IP주소가 있는 컴퓨터의 인터넷에서 구동하는 코드를 작성하면 `유입되는 요청` 을 처리하고 활용할 수 있는 서버를 만드는 코드를 작성할 수 있는것이다.

물론 Node.js가 필수는 아니고 `PHP / asp.net / Ruby on Rails` 등 다른 대체언어도 있다.

사용자 입력 유효성 검사, 독자적인 데이터베이스 서버에서 구동하는 데이터베이스와 소통하되 일반적으로 백엔드에서, 즉, `서버 사이드 코드` 에서 접근하며 클라이언트로 `다시 응답` 을 보내게 된다.
이 응답은 `HTML Text / HTML Code` 로 구성될 수 있고, 이후 클라이언트를 통해 처리되나 `파일 / JSON / XML` 을 비롯한 다른 종류의 데이터일 수도 있다.

이 응답은 단순한 콘텐츠 그 이상이며 `응답 / 요청` 은 모두 `헤더` 를 지니고 있다.
이는 요청과 응답에 첨부되는 일종의 `메타 정보` 로 내용이 무엇인지를 설명해준다.

#### Node.js로 서버를 구성하게 될 코드

![](https://velog.velcdn.com/images/chromeheartz/post/dcb4454b-c1d8-4f54-8c08-87889884e941/image.png)

`요청 / 응답` 전송이 일부 프로토콜, 즉 `표준화` 된 소통 방식을 통해 이뤄진다.
브라우저가 작업할 수 있는 응답을 회신하기 위해 사용하는 프로토콜인 `HTTP / HTTPS` 를 통해 정의되는 규칙들을 준수해야 한다.

`HTTP` 는 하이퍼텍스트 전송 프로토콜의 약어이고, 이를 통해 유효한 요청이 어떤 형태를 지니고 어떤 데이터가 브라우저에서 서버 혹은 그 반대로 전송되어야 할지를 정의한다.
`HTTPS` 는 기본적으로 HTTP와 동일하며 여기에 전송된 모든 데이터가 실제로 암호화되는 `SSL 암호화` 가 켜져있어 누군가 연결을 스푸핑하더라도 데이터를 읽을 수 없도록 한다.
