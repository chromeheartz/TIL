## NodeJS

### 01. Node.js란 무엇인가

---

### 📌 Node.js란?

![](https://velog.velcdn.com/images/chromeheartz/post/e7d482fa-819b-490e-8c03-b1f1b21441ef/image.png)

`Node.js` 는 `Javascript Runtime` 이다.
Javascript는 브라우저에서 많이 사용하는 프로그래밍 언어로 도메인을 편집하거나, 브라우저에서 로딩된 페이지를 편집하거나, 팝업 / 모달 또는 효과를 불러오는 데에 사용한다.
`Javascript` 는 브라우저에서 실행되는 언어로 로딩 후에도 페이지와 `상호작용` 이 가능하게 해준다. 따라서 인터랙티브 사용자 인터페이스를 구현하는데 필수적인 요소이다.

그러나 Javascript의 기능은 이게 전부가 아니다. Node.js는 `다른 버전의 Javascript` 라고 할 수 있다. 자바스크립트 기반으로 만들어졌으며, 몇몇 기능을 더하고 기존 Javascript로 브라우저에서 가능했던 기능 중 일부는 빠졌다.

> 즉, `Javscript` 를 다른 환경에서 구현한 것이라고 할 수 있다.
> 자바스크립트 코드를 서버에서 실행할 수 있도록 해주며 이론적으로는 서버뿐만이 아니라 어떤 기기에서도 실행할 수 있다.
> ✅ Javascript를 브라우저뿐만 아니라 다른 곳에서도 평범한 프로그래밍 언어처럼 실행할 수 있도록 해준다.

따라서 인터넷에서 작동하는 컴퓨터라고 할 수 있는 `서버` 에서 웹 애플리케이션을 구현하는데 매우 유용하다. 이는 `Node.js` 를 사용해 Javscript를 브라우저 밖에서도 실행할 수 있음을 뜻한다.

#### 기술적으로 어떻게 가능할까?

![](https://velog.velcdn.com/images/chromeheartz/post/39a97aeb-c7af-43f9-aa5f-10e4a4cd6116/image.png)

`Node.js` 는 `V8` 을 사용한다. `V8` 은 구글에서 개발한 `Javascript 엔진` 으로 Javscript를 브라우저에서 실행해 준다.

#### 그럼 `엔진` 이란 무엇일까?

엔진은 브라우저에서 실행중이거나 `V8` 에서 구현했다면 노드에 있는 `Javascript 코드 / Node.js Javscript` 코드를 `Machine Code` 로 컴파일한다.
브라우저도 같은 일을 하는데 V8은 브라우저에서 Javscript 코드를 가지고 머신 코드로 컴파일해준다. 이 작업은 V8에 의해 이루어지고 V8자체는 `C++` 코드로 쓰여있지만 이는 나에게 중요하지 않다.

즉, Node.js는 `V8 코드베이스` 를 사용하며 `로컬 파일 시스템 활용 / 파일 열기 / 파일 읽기 / 파일 삭제` 등의 기능을 추가해준다. 이러한 작업은 브라우저에서 불가능하다.

보안상의 이유로 브라우저에서는 로컬 파일 시스템에 액세스할 수 없으므로 이러한 기능은 지원하지 않는다. `Node.js` 는 브라우저에서 실행되지 않기 때문에 이러한 제한이 없는 것이다.

브라우저 Javascript보다 더 다양한 작업을 가능하게 해주는 기능들을 Javscript 엔진에 추가해준다.

> 📍 짚고 넘어가야할 중요한 점은 일부 기능은 빠진다는 것이다.
> 브라우저에서는 Javscript를 사용해 DOM(문서 객체 모델)과 상호작용하게 된다. 이러한 기능들이 제외가 되는 것이다.
