## NodeJS

### 03. REPL 작업 vs 파일 사용

---

### 📌 노드 코드를 실행하는 두 가지 다른 방법

![](https://velog.velcdn.com/images/chromeheartz/post/a4b1860f-f047-49cf-8abf-c5e385bd012a/image.png)

첫 번째 방법은 `REPL` 이다.
`REPL` 은 사용자 입력값을 읽고, 입력값을 평가하고, 결과값을 출력하고 돌아가서 새로운 입력값을 기다리는 과정을 나타낸다.
`Terminal` 에 `node` 를 입력하면 `REPL` 을 사용하게 된다.
여기서

```
const fs = require('fs')
fs.writeFilseSync
```

이런식으로 작성할 수 있다. 하지만 코드가 파일로 저장되지 않고, 노드 애플리케이션을 한 줄씩 작성하게 된다. 이 코드들은 독립적으로 기능하지 않는다. 또한 `Cmd + c` 로 프로세스를 종료하면 그대로 끝이다. 어디에도 저장되지 않으며 다시 실행할 수 있는 파일이 아니다.
`REPL` 을 대체하도록 파일을 실행하자.

![](https://velog.velcdn.com/images/chromeheartz/post/19635589-ba6b-424d-8f12-c2a32039465d/image.png)

파일을 사용하게 되면 코드를 미리 작성해서 항상 실행할 수 있다는 점이다.
그리고 실제 앱을 만들 때는 코드를 파일로 저장해서 배포하고 다른 개발자들과 공유하거나 작업을 멈추고 나중에 다시 돌아오더라도 코드를 잃지 않을 수 있다.

> `REPL` 은 명령을 실행해서 특정 기능을 시험해보기는 좋다. 실제 애플리케이션을 만들기보다는 일부 기능을 시험할때 사용하면 좋을 것이다.
