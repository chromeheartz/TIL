### 📌 Node.js 역할과 사용법

![](https://velog.velcdn.com/images/chromeheartz/post/87ef9ed8-1cd7-478e-b21e-dcf00e9a5c73/image.png)

이제 Node.js로 `서버 사이드 코드` 를 작성하고 서버에서 실행할것이다.
이를 위해 전체적인 그림을 살펴보아야 한다.

먼저 사용자가 있는데, 클라이언트 컴퓨터 / 휴대전화로 브라우저에 접속하거나 모바일 앱을 사용할 수도 있다. 어떻게 이들과 소통하는지는 이후에 보고, 일단 기존의 브라우저만을 살펴보자.

브라우저를 사용하는 사용자가 있고, `HTML / CSS / JAVASCRIPT` 를 사용해서 웹 페이지를 만들 수 있는데, `my-page.com` 페이지에 방문해서 해당되는 요청을 보낸다.
예를 들어 브라우저에 URL을 입력함으로써 URL에 요청을 전달한다.

여기서 서버가 등장하는데 서버는 인터넷에서 실행 중인 컴퓨터로 해당 도메인과 관련된 IP를 가지고 있으며 자동으로 할당된다. 그리고 서버에서 들어오는 요청을 처리해서 응답을 회신하는 코드를 실행하고자 한다.

무조건은 아니지만 응답의 대부분의 경우 브라우저가 표시할 수 있는 HTML일 것이다.
물론 CSS 파일 / 서버가 아닌 브라우저용 Javscript 코드가 포함된 파일일 수도 있다.

서버에서는 일반적으로 성능 또는 보안상의 이유로 브라우저에서 할 수 없거나 하고 싶지 않은 작업을 수행한다. 데이터를 불러오거나 저장하기 위해 `데이터베이스에 접속` 하거나 `사용자 인증` 을 수행하기도 한다.

이는 보안을 강하게 하고 해킹을 막기 위해 사용자가 접속할 수 없는 곳에서 진행해야 한다.
사용자가 올바른 이메일 주소를 입력했는지 확인하는 등의 `입력값 검증` 에도 사용된다.

브라우저는 항상 속일 수 있다. 사용자가 `브라우저 사이드 코드` 를 편집할 수도 있다.
하지만 서버는 유저가 액세스할 수 없으므로 안전해서, 일반적으로 서버 운영에는 `비즈니스 로직` 이 적용된다.

사용자에게 데이터를 회신하는 코드를 서버에 작성해서 클라이언트가 사용하게 할것이다.

#### 서버에만 국한되지 않는다

![](https://velog.velcdn.com/images/chromeheartz/post/f3b2ac0b-07c2-4cd9-a3aa-bed3dfb6aa7e/image.png)

> ✅ Node.js는 서버의 코드 실행에만 국한되지 않는다.
> Javascript 런타임이며 브라우저가 필요하지 않고, 서버를 구축하지 않고도 실행할 수 있다.
> 예를 들어 `로컬 유틸리티 스크립트 / 빌드 툴` 같은 것들에도 사용할 수 있다. React, Angular, Vue등을 사용해보면 이러한 언어 또는 프레임워크에 필요한 모든 빌드과정에 간접적으로 Node.js를 사용했다는 뜻이다.

파일을 읽고, 쓰고, 편집할 수 있도록 파일 시스템에 액세스하면 컴퓨터에서 대중에 노출되지 않은 채로 다양한 유틸리티 작업을 수행할 수 있다.

#### Node.js Role

![](https://velog.velcdn.com/images/chromeheartz/post/550f4e48-17bc-456e-8537-6647515558fa/image.png)

일반적으로 Node.js를 가장 흔히 활용하는 용도는 `웹 개발 / 서버 사이드 코드` 이다.
그리고 이는 `PHP` 와의 중요한 차이점이 된다.

예를 들어 Node.js의 경우에는 서버에서 작동하는 코드를 작성하기만 하는 것이 아닌 `서버 자체` 를 스스로 작성하게 된다. 따라서 코드가 들어오는 요청을 처리해서 다른 코드로 보내준다.
PHP에서는 Apcache나 Nginx와 같은 추가 툴이 있어 요청을 받아 PHP를 실행해준다. 하지만 Node.js는 둘 모두를 수행하는 것이다.

이는 매우 중요한 점으로 모든 비즈니스 로직을 운영하는데 사용하기도 한다.
들어오는 요청을 처리하는 것뿐만 아니라 요청받은 데이터를 처리하고, 파일을 활용하고, 데이터베이스를 활용하는 등의 흥미로운 일들을 할 수 있다.

또한 `응답 사이드` 도 처리하는데, 들어오는 요청만이 아니라 Node.js를 사용해서 클라이언트에게 `데이터를 회신` 할 수도 있다.
`HTML / 동적 콘텐츠 / json 형식 / xml / 일반 파일 ...` 이 가능하다.

### 📌 Node.js의 큰 장점

프론트엔드에서 빌드 툴에 이르기까지 현대 웹 개발에서 널리 사용되는 `Javscript` 를 사용한다는 점이다. 이를 서버사이드에서 사용할 수 있다면 다른 언어를 여러 가지 배울 필요가 없다.
하나의 동일한 언어를 서버 사이드 코드에서도 사용할 수 있다.

효율적일 뿐만 아니라 성능이 우수하고 널리 사용되는 언어이다.
