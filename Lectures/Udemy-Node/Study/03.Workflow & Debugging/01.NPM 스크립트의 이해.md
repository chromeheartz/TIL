## Workflow & Debugging

### 01. NPM 스크립트의 이해

---

### 📌 NPM 스크립트

이전에는 `node app.js` 를 실행해야 파일과 앱 내지는 노드 애플리케이션을 시작할 수 있었다. 그리 부담되는 일은 아니지만 `Node.js` 프로젝트에 몇 가지 스크립트를 정의하여 이런 태스크에 있어 도움을 받는것도 가능하다.

`NPM` 을 사용해야 하며 이는 `Node Package Manager` 를 뜻한다.
Node.js 내부에 함께 설치되어 있다. 별도의 작업 없이도.
이제 NPM을 사용해서 패키지에 몇몇 `제 3자 패키지` . 노드 코어에 포함되지 않은 패키지들을 설치할 수 있다.
`npm init` 으로 초기화를 시킬 수 있다.
이렇게 명령어를 입력하면 프로젝트의 기본 설정을 하라는 것을 볼 수 있다.
이렇게 하면 `package.json` 파일이 생기게 되며 설정한 모든 설정 및 구성을 확인할 수 있다.

기본적으로 Javascript 객체와 매우 유사해 보이는 특수한 종류의 데이터 형식인 `JSON` 형식을 사용하며 이를 기반으로 한다.

#### 이해하고 넘어가야할 점

키들은 항상 쌍따옴표 안에 입력하고 숫자 / 배열 / boolean을 제외한 다른 값들도 쌍따옴표안에 입력해야 하지만, 일단 무시하고 넘어가자
우선은 현재 파일 프로젝트의 `구성 파일` 이라고만 이해하자.

#### 구성 파일은 어떤 역할을 할까?

구성파일을 살펴보면 `scripts` 구획이 있는데 지금은 아무 역할도 하지 않는 `script` 가 기본값으로 있고, 여기에 다른 스크립트를 추가해서 실행할 수 있다.
`start` 라고 키값에 넣어보자. 이는 특수한 스크립트 이름이다.
`"start": "node app.js"` 를 입력해보자.
이후에 `npm start` 명령어를 실행하면 할당된 이름으로 항상 start 스크립트를 탐색할 것이다. 즉, 예전과 동일한 일을 했지만 `node app.js` 가 실행되는 것이다.

이제 간단히 `npm start` 만 실행하면 되고, 어느 Javascript파일이 `진입 파일` 인지를 일일이 파악하지 않아도 된다.

특별한 스크립트(start)이외에도 이름을 자유롭게 선택해서 추가할 수 있다.
예를 들어`start-server` 가 있는데 이것도 `node app.js` 를 실행할 수 있다.

![](https://velog.velcdn.com/images/chromeheartz/post/bf0745dd-3730-4661-a219-6d409976be1f/image.png)

하지만 정작 실행을 해보면 `알려지지 않은 명령어` 라는 오류가 등장하는데, 왜냐하면 여기에 스크립트 이름만 입력했을 때는 작동하지 않기 때문이다.
`start` 는 특수한 경우이다.

> 📌 따라서 `개별 설정된 이름` 이 있는 일반 스크립트의 경우는 `npm run` 을 붙여서 시작시켜주어야 한다.

사실 이렇게 되면 `npm run start-server` 는 불필요하며 `npm start` 가 훨씬 짧지만 본인만의 스크립트를 추가할 수 있다는 것을 짚고 넘어가면 된다.

`Angular, React, Vue` 또는 기타 최신 프론트엔드 개발 워크플로우로 작업을 해보았다면 자주 이런 스크립트를 사용해서 프로젝트의 빌드 워크플로우를 시작하는 경험을 해보았을테고, 실행하고 싶은 다양한 태스크에 이를 적용할 수 있다.
