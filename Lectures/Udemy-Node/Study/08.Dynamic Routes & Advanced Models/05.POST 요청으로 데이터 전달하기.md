### 📌 POST 요청으로 데이터 전달하기

`product-detail.ejs` 에 있는 `add to cart` 버튼을 `form` 으로 감싸서 `method` 를 `post` 로 지정해놓았다.

지금까지는 `url` 에서 정보를 가져와서 제품 정보를 표시하는 방법을 보았다.
이제 장바구니 버튼으로 데이터를 포스팅할 때 어떤 제품을 장바구니에 추가할지 확인해야 한다.

> ✅ `post` 요청에는 중요한 사실이 있는데 `request body` 를 통해 데이터를 전달할 수 있다. 이는 `get` 요청에서는 불가능하다.

제품을 추가하기 위한 폼이 있고, 이를 추가할 때 언급한 대로 이는 `자동으로 모든 입력 데이터가 본문에 포함된 요청` 을 준다.
데이터를 포스팅할때는 그냥 정보를 `post 본문` 에 넣으면 된다.
따라서 `product-detail` 에는 `post` 요청의 일부로써 일부 중요 정보를 입력해야 한다.

```html
<form action="/cart" method="post">
  <button class="btn" type="submit">Add to Cart</button>
  <input type="hidden" name="productId" value="<%= product.id %>" />
</form>
```

일단 `input` 의 타입을 `hidden` 으로 바꾸어서 페이지에 실제로 표시되지는 않더라도 전송되는 요청에는 인코딩 되도록 바꾸어준다.
`value` 에는 `productId` 에 대해 요청에 추가되는 값이 된다.
이 `id` 를 `node` 에 보내야하는데 `url` 을 사용하지 않고 `post` 요청에 대한 데이터를 전달해보자.

`shop.js` 에서 `router.post('/cart', shopController.postCart);`
새로운 라우트를 추가한다.

```js
// controllers/shop.js
exports.postCart = (req, res, next) => {
  const prodId = req.body.productId;
  res.redirect("/cart");
};
```

추가로 파일에서 product를 가져와서 `cart` 에 추가하도록 한다.
요청에 액세스해서 `prodId` 를 추출하려는데 요청 본문의 일부인 `req.boduy` 에 액세스하면 가져올 수 있다.

이후 `/cart` 로 리다이렉트를 해줄것인데 이렇게 하면 Cart페이지가 렌더링된다.
또한 버튼을 눌러보면 무작위한 id가 기록되는 것을 볼 수 있다.

또한 버튼 부분을 많은곳에서 사용하므로 `add-to-cart.ejs` 라는 이름으로 바꾸어주고 해당 버튼이 들어가는 부분에 `include` 를 사용해서 구문을 교체했다.
