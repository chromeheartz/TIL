### 📌 상품 상세 데이터 로드하기

이제 동적세그먼트를 출력할 수 있게 되었으므로 정보를 사용해서 product파일에서 해당 제품을 불러올 수도 있다. product와의 상호작용을 담당하는 `product model` 을 통하게 된다.
그리고 이러한 제품 세부 정보를 보여주는 뷰를 회신할 수 있다.

```js
  static findById(id, cb) {
    getProductsFromFile(products => {
      const product = products.find(p => p.id === id);
      cb(product);
    });
  }
```

`findById` 로 `static` 메소드를 추가한다.
id와 제품 검색을 마치면 실행할 콜백이 인수로 들어간다. 여기 함수에서 모든 제품을 가져오기 때문이다. 파일 전체를 읽어야함.
여기에 제품 하나에 대한 쿼리를 실행할 수 있는 데이터베이스가 업슨ㄴ데, 이후에 데이터베이스를 추가하고나면 할 수 있다.

지금은 `fetchAll` 과 같은 모든 제품을 가져오게 된다.
이를 회신하기 전에 콜백에서 원하는 ID를 가진 제품 하나를 필터링 하려고 한다.
여기서 회신하는 `products` 는 이미 파싱된 제품 배열로 제품마다 id가 있고, 할당 후 저장한다.

따라서 각 객체가 ID를 가치고 있는 객체 배열을 얻게 되며, 일반적인 javascript코드를 사용해서 관심있는 제품을 필터링 할 수 잇다.
원하는 product를 찾아서 `임시 변수` 에 저장하면 된다.
`find` 메소드를 사용해서 찾아낸다.

이제 함수를 추가했으므로 `shop.js` 컨트롤러에서 이를 사용할 수 있다.

```js
// controllers/shop.js
exports.getProduct = (req, res, next) => {
  const prodId = req.params.productId;
  Product.findById(prodId, (product) => {
    console.log(product);
  });
  res.redirect("/");
};
```

여기서 `Rroduct` 클래스가 임포트됨을 기억해야 한다.
`Product.findById` 이렇게는 볼 수없다. 이는 비동기성함수이기 때문에, 콜백을 입력해야 한다.

함수에서 올바르게 가져오는지를 확인해보기 위해 `console.log` 로 찍어보게 되면 제대로 작동하는 것을 알 수 있다.
