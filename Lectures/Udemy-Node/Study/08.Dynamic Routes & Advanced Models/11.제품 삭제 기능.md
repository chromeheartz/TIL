### 📌 제품 삭제

이제 `제품 삭제` 를 추가 해보자.
먼저 `admin/product.ejs` 파일에 POST 요청을 보내 제품을 삭제하도록 `form` 에 `delete` 버튼이 이미 랩핑되어있다. 따라서 해당 라우트가 출발점이 될 수 있다.
`router.post('/delete-product', adminController.postDeleteProduct);`
POST요청이기 때문에 `URL 경로의 정보` 를 암호화하지 않고 요청 본문의 일부로 넣으면 된다.

```html
<form action="/admin/delete-product" method="POST">
  <input type="hidden" value="<%= product.id %>" name="productId" />
  <button class="btn" type="submit">Delete</button>
</form>
```

name을 이용해 정보를 추출할 수 있게 해준다.
이제 컨트롤러를 만드는데

```js
exports.postDeleteProduct = (req, res, next) => {
  const prodId = req.body.productId;
  Product.deleteById(prodId);
  res.redirect("/admin/products");
};
```

간단하게 만들어줄 수 있다.
`productId` 를 통해 정보를 얻어서 삭제 메소드를 연결해주면 된다.

```js
// models/product.js
  static deleteById(id) {
    getProductsFromFile(products => {
      const product = products.find(prod => prod.id === id);
      const updatedProducts = products.filter(prod => prod.id !== id);
      fs.writeFile(p, JSON.stringify(updatedProducts), err => {
        if (!err) {
          Cart.deleteProduct(id, product.price);
        }
      });
    });
  }
```

먼저 제품 배열중에 어떤 제품을 삭제할지 알아야 하니 `find` 메소드를 통해 찾아낸다.
이제 제품 배열에서 제거하는 업데이트를 할것인데 제일 빠른 방법은 `updatedProducts` 라는 새로운 상수를 만들어 `filter` 메소드를 사용하는 것이다.
`filter` 역시 익명함수로 함수의 반환기준에 맞지 않는 모든 요소를 새로운 배열로 반환한다.
따라서 기준에 부합하면 요소가 유지되기 때문에, 삭제하려는 `id` 와 다른 `id` 를 가진 모든 요소를 유지해서 새로운 배열로 쓰이게끔 구현해준다.

마지막으로 콜백 함수를 통해 오류가 있는지 확인해준다.
