### 📌 쿼리 매개변수 사용

장바구니에 대해 좀 다루어봤는데 아직 몇몇부분들이 빠져있다.
예를 들어 `Cart` 를 표시하지도 않고, 항목을 지울 수도 없는 상태지만 일단 넘어가도록 한다. 일단 `상품 편집` 에 집중

`edit-product.ejs` 파일에서 최종적으로 하고싶은것은, `add-product` 와 동일한 렌더링 방식을 사용하는 것이다.
차이점은 `내가 편집하고 싶은 상품의 값` 으로 미리 채우고싶다는 점이다.

최종적으로는 `동일한 HTML` 을 쓴다면 동일한 템플릿을 재사용하는게 합리적일 것이다.
`add-product.ejs` 는 삭제하고 `edit-product.ejs` 로 같이 사용하게 하기 위해 `admin.js` 에서의 `getAddProduct의 render` 메소드를 수정해주어야 한다.

같은 템플릿으로 사용할 수 있게 `path` 를 바꾸어주고 Add Product 네비게이션을 클릭하면 잘 작동한다.
추가로, 상품을 편집하는 경우(edit) 또한 만들어준다.
차이점은 `상품 정보` 로 전달하는 것의 차이가 있을 것이다. 그러나 문제는 `컨트롤러 활동` 에 어떻게 도달할 수 있느냐이다.

`Edit` 버튼이 있으니, 클릭하게 되면 그냥 `edit-product` 를 호출하는 대신 추가하고싶은 상품의 ID 또한 덧붙이면 된다.
그래서 `products.json` 파일에서 실제 ID를 가져오고 URL에 반영하고 로드하면 이제 상품에 대한 데이터로 형식을 미리 채워놓고 싶다.
또한 Save버튼을 누르면 상품을 `생성` 하는 것이 아니라 `기존의 것을 편집` 하게 하고 싶다 이를 위해 2가지가 필요하다.

> `전달해야할 ID` / 상품을 생성하는 대신 `편집하고 싶은 정보`

`admin routes` 로 가보면 새로운 라우트를 추가할 수 있는데, 여기에 `edit-product` 라우트에 ID를 추가해준다.
`router.get('/edit-product/:productId', adminController.getEditProduct);`
`동적 경로 세그먼트` 를 넣어주어야 한다.

이제 상품을 편집하고 싶기 때문에, `editing` 에 추가 정보 필드를 전달하고 `true` 로 설정해서 `if` 조건으로 확인할 수 있도록 한다.

`Save` 를 클릭하는 시점에 상품을 추가ㅏㅎ고 해당 라우트로 요청을 발송하던가 편집해서 다른 라우트를 통해 발송할 수 있다.
하지만 URL에서 쿼리 매개 변수를 사람들이 전달해 줘야 하게 만듦으로써 추가 확인을 획득하고 싶다고 가정해보자.
`title=new` 같은 다수의 쿼리 매개변수를 입력할 수 있는데 이를 `optional data` 라고 한다. 도달하게 되는 이 라우트는 `?` 표시까지의 부분을 통해 결정된다.

따라서 `routes` 파일로 전달할 쿼리 매개 변수에 대한 정보는 추가할 필요가 없다.
이 경로들은 영향받지 않는데, 항상 컨트롤러에 있는 쿼리 매개 변수는 확인할 수 있따.

> 따라서 `editMode` 라는 변수를 만들고 `쿼리 객체가 있는가` 를 확인할 수 있다.

```js
exports.getEditProduct = (req, res, next) => {
  const editMode = req.query.edit;
  if (!editMode) {
    return res.redirect("/");
  }
  const prodId = req.params.productId;
  Product.findById(prodId, (product) => {
    if (!product) {
      return res.redirect("/");
    }
    res.render("admin/edit-product", {
      pageTitle: "Edit Product",
      path: "/admin/edit-product",
      editing: editMode,
      product: product,
    });
  });
};
```
