### 📌 handlebars

다른 뷰 엔진을 살펴볼것인데 `breaking change` 를 만들것이므로, 버전을 고정시켜서 진행해보자.

`npm install --save express-handlebars@3.0`

`pug` 를 먼저 보았는데 이제 볼것은 `Handlebars` 이다.
평범한 `html` 방식으로 `일반 html / 템플릿 논리` 가 섞여있다.
`Handlebars` 를 이용하기 위해서는 `app.js` 로 가서 `view engine` 을 변경해야 한다.

`const expressHbs = require('express-handlebars');`
이렇게 불러온 이후에 이러한 엔진을 사용 가능하도록 `app.engine()` 메소드를 호출해서 넣어준다.

```js
// app.js
const expressHbs = require("express-handlebars");

const app = express();
app.engine("handlebars", expressHbs());
app.set("view engine", "handlebars");
```

함수가 초기 설정된 뷰 엔진을 반환해서 `engine` 에 넣는것이다. `express handlebars` 가 이 엔진을 제공하면 `view engine` 을 `handlebars` 로 변경한다.

우선 `404.handlebars` 페이지 먼저 만들어보자.

`app.engine('hab', expressHbs())` 이런식으로 만들면 `404.hbs` 로 파일을 만들어도 된다.
템플릿에 데이터를 전달하는 방식은 엔진 별로 별로 다르지 않다 대부분 `객체에 키-값` 쌍을 넣어서 키와 값을 템플릿에서 사용할 수 있다. 출력하는 방식만 엔진별로 다른것이다.

`handlebars` 에서는 `<title>{{ pageTitle }}</title>`
이런식으로 중괄호 두 개를 열고 닫아서 사용한다.

```html
// shop.hbs
<main>
  {{#if hasProducts }}
  <div class="grid">
    {{#each prods}}
    <article class="card product-item">
      <header class="card__header">
        <h1 class="product__title">{{ this.title }}</h1>
      </header>
      <div class="card__image">
        <img src="src" alt="A Book" />
      </div>
      <div class="card__content">
        <h2 class="product__price">$19.99</h2>
        <p class="product__description">
          A very interesting book about so many even more interesting things!
        </p>
      </div>
      <div class="card__actions">
        <button class="btn">Add to Cart</button>
      </div>
    </article>
    {{/each}}
  </div>
  {{ else }}
  <h1>No Products Found!</h1>
  {{/if}}
</main>
```

`조건부` 또한 `{{}}` 구문으로 만들어낼 수 있다.
하지만 `if` 문으로 끝나는 문장을 만들려면 `해시태그` 로 시작해야 하고 `{{/if}}` 로 닫아주어야 한다. `hbs` 는 `prods.length > 0` 같은 문장을 지원하지 않고, `true / false` 를 출력하는 키만 지원한다.

### 📌 레이아웃

`pug` 와는 조금 다르게 `layout` 을 만들려면 `handlebars` 엔진을 등록하는데 몇 가지 옵션을 전달해주어야 한다.

```js
// app.js
app.engine(
  "hbs",
  expressHbs({
    layoutsDir: "views/layouts/",
    defaultLayout: "main-layout",
    extname: "hbs",
  }),
);
app.set("view engine", "hbs");
app.set("views", "views");
```

`layout` 의 디렉토리 경로를 지정해주고, `기본 레이아웃` 의 파일을 정의해주어야 한다.
또한 `extname` 이라는 전체 파일이 아닌 레이아웃에만 적용되는 확장명을 설정해서 레이아웃을 제외한 모든 파일에 이 부분이 적용 되게 해주어야 한다.

```html
<!-- main-layout.hbs -->
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <meta http-equiv="X-UA-Compatible" content="ie=edge" />
    <title>{{ pageTitle }}</title>
    <link rel="stylesheet" href="/css/main.css" />
    {{#if formsCSS}}
    <link rel="stylesheet" href="/css/forms.css" />
    {{/if}} {{#if productCSS}}
    <link rel="stylesheet" href="/css/product.css" />
    {{/if}}
  </head>

  <body>
    <header class="main-header">
      <nav class="main-header__nav">
        <ul class="main-header__item-list">
          <li class="main-header__item">
            <a class="{{#if activeShop }}active{{/if}}" href="/">Shop</a>
          </li>
          <li class="main-header__item">
            <a
              class="{{#if activeAddProduct }}active{{/if}}"
              href="/admin/add-product"
              >Add Product</a
            >
          </li>
        </ul>
      </nav>
    </header>
    {{{ body }}}
  </body>
</html>
```
