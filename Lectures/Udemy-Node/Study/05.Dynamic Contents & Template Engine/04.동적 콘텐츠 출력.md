### 📌 동적 콘텐츠 출력

`Shop 템플릿` 은 렌더링하고 있지만 `동적 콘텐츠(products)` 는 렌더링 하지 않는다.
사실은 이것이 이 모듈의 전체 개념이다.

관리자 데이터를 결과와 함께 가져오는데 실제로 관리자 데이터 결과에서 결과를 가져ㅘ서 템플릿으로 전달한 뒤 투입함으로써 템플릿 파일에서 풀력하도록 해야한다.
`렌더링 메소드` 는 `view` 에 추가되어야 하는 데이터를 전달할 수 있게 한다.
이를 위해 렌더링 메소드에 두 번째 인수를 전달할 수 있는데 `view` 에 추가되어야 하는 데이터를 전달할 수 있다.

```js
// shop.js
router.get("/", (req, res, next) => {
  const products = adminData.products;
  res.render("shop", { prods: products, docTitle: "Shop" });
});
```

객체 형태로 전달하며 이 객체를 키 이름에 매핑한 뒤 템플릿 내부에서 전달하는 데이터를 참조하기 위해 사용하고, 이름의 혼동을 피하기 위해 `prods` 라는 이름을 사용해서 전달한다.
이건 템플릿으로 전달되고 이제 템플릿 내에서는 prods에만 접근할 수 있게 된다.
예시로 `docTitle` 도 같이 전달해보자.

#### `#와 {}`

`#` 와 중괄호 2개를 사용하고 해당 중괄호 사이에 `view` 로 전달하는 값을 무엇이든 넣으면 된다.
`title #{docTitle}`

이제 `prods` 가 있는지 없는지 조건에 따라서 사용자에게 보여주는 부분을 다르게 구현해주면 된다.

```html
<!-- shop.pug -->
<!DOCTYPE html>
html(lang="en") head meta(charset="UTF-8") meta(name="viewport",
content="width=device-width, initial-scale=1.0")
meta(http-equiv="X-UA-Compatible", content="ie=edge") title #{docTitle}
link(rel="stylesheet", href="/css/main.css") link(rel="stylesheet",
href="/css/product.css") body header.main-header nav.main-header__nav
ul.main-header__item-list li.main-header__item a.active(href="/") Shop
li.main-header__item a(href="/admin/add-product") Add Product main if
prods.length > 0 .grid each product in prods article.card.product-item
header.card__header h1.product__title #{product.title} div.card__image
img(src="경로") div.card__content h2.product__price $19.99
p.product__description A very interesting book about so many even more
interesting things! .card__actions button.btn Add to Cart else h1 No Products
```

`.grid` 처럼 아무것도 없는 경우 `div` 로 자동으로 간주되기 때문에 붙일 필요가 없다.

#### each

`each` 키워드를 추가함으로써 루프를 생성한뒤 값을 추가해주며 이 값은 현재의 반복을 위해 저장해야 한다. 따라서 현재 기점에서는 단일 결과가 될 것이고 `in` 을 적어준 뒤 루프하길 원하는 배열을 입력해준다.

![](https://velog.velcdn.com/images/chromeheartz/post/bc986bf5-7bba-4b9e-82b1-934e384de02a/image.png)

`first book` 을 `add-product` 로 만들어보게 되면 출력이 되는 것을 볼 수 있다.

> ✅ 템플릿에서만 뭔가 변경했다면 서버를 재시작할 필요도 없고 `nodemon` 도 작동하지 않을텐데 왜냐하면 템플릿은 `서버측 코드` 에 포함되지 않기 때문이다. 단순히 상황에 따라 선정되는 요소들에 불과하다.
> 따라서 다음번에 들어오는 요청을 위해 템플릿을 변경하는 경우 새로운 버전을 자동으로 받아들인다.
