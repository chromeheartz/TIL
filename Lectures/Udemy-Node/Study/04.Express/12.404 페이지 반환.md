### 📌 404 페이지

`views` 에 새로운 파일을 추가해보자.
`404.html` 로 만들것인데 `Page Not Found` 를 출력해주는 페이지로 사용할것이다.

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <meta http-equiv="X-UA-Compatible" content="ie=edge" />
    <title>Page Not Found</title>
  </head>
  <body>
    <h1>Page Not Found!</h1>
  </body>
</html>
```

이제 이 파일을 회신할것인데, 먼저 `path` 모듈의 도움을 받아서 경로를 구축해서 `모든 운영 체제` 에서 작동하도록 해야 한다.

```js
// app.js
app.use((req, res, next) => {
  res.status(404).sendFile(path.join(__dirname, "views", "404.html"));
});
```

기존과 조금 다른점은 `../` 이 필요가 없다는 점이다.
이미 app.js파일을 보고 있고, 프로젝트 폴더에 있고, 루트 폴더에 있으므로 상위 경로로 올라갈 필요가 없다.
이 상태에서 라우트가 없는 무작위 페이지를 입력하면 `404.html` 이 잘 회신되고 있음을 볼 수 있다.
