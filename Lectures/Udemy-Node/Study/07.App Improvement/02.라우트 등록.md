## App improvement

### 02. 라우트 등록

---

### 📌 라우트 등록

라우트 등록을 위해 `routes` 를 확인해보자.
admin/ shop이 있고, 적절하게 구분해놓긴 했다.

이제 `/products` 에 대한 `get` 라우트도 필요하다. `navigation.ejs` 를 수정했기 때문에 새로운 링크가 필요하다.
또한 `/cart, /checkout` 도 필요할것이니 3개를 다 추가해주자.

```js
router.get("/products");

router.get("/cart");

router.get("/checkout");
```

또한 `admin.js` 에 들어가서 새로운 get라우트를 추가한다.
여기서 포인트는 `admin` 또한 다른 라우트처럼 이미 `/admin` 으로 시작하는 경로안에 있다는 것이다. `app.js` 에서 그렇게 필터링 한것이다.
물론 이에 대한 컨트롤러 함수도 필요하다.

`router.get('/products')`

#### 컨트롤러는 어떻게 하면 될까?

거기에는 다른 접근법을 적용해보자. 대부분의 항목은 여ㅛ전히 제품과 관련되어있다.
그러나 `장바구니 / 결제` 와 같은 새로운 라우트도 있다.
나중에는 주문 내역도 추가할 예정인데, 물론 이들은 간접적으로 제품과 연관되어 있고 최종적으로 제품을 구매하게 된다.

이들은 의미론적 또는 논리적 관점에서 볼 때는 일반적인 매장 개념에 속한다고 할 수 있다.
따라서 기존에 사용하고 있던 controllers에서 products를 shop.js로 바꾼다. 즉, `shop 컨트롤러 / admin 컨트롤러` 를 새로 추가해서 논리를 두 부분으로 나눈다. 물론 이렇게 하는 대신에 `products 컨트롤러 / shop 컨트롤러` 를 사용해서 제품과 관련된것은 `products` 에 두고 장바구니나 계산대 같은것은 `shop` 에 둘 수도 있다.

현재 진행한 방식대로 하면 admin관련 라우트인 `getAddProduct / postAddProduct` 를 `shop.js` 파일에서 삭제하고 `admin.js` 파일로 옮기는게 더 말이 된다.
여기에도 모델이 필요하기 대문에 임포트를 추가한다.

따라서 `routes/admin.js` 에서도 `shop` 이 아닌 `admin` 으로 경로를 바꾸어주는 작업들을 해야한다.

`shop.js` 에서부터 시작해보자.
먼저 `getIndex` 함수를 만들어서 인덱스 페이지를 렌더링해야하므로 `미들웨어 함수` 를 가져온다. 물론 제품을 불러오고 뷰를 렌더링해야한다.

```js
// shop.js
const Product = require("../models/product");

exports.getProducts = (req, res, next) => {
  Product.fetchAll((products) => {
    res.render("shop/product-list", {
      prods: products,
      pageTitle: "All Products",
      path: "/products",
    });
  });
};

exports.getIndex = (req, res, next) => {
  Product.fetchAll((products) => {
    res.render("shop/index", {
      prods: products,
      pageTitle: "Shop",
      path: "/",
    });
  });
};

exports.getCart = (req, res, next) => {
  res.render("shop/cart", {
    path: "/cart",
    pageTitle: "Your Cart",
  });
};

exports.getCheckout = (req, res, next) => {
  res.render("shop/checkout", {
    path: "/checkout",
    pageTitle: "Checkout",
  });
};
```

`shop.js` 에 `index, cart, checkout` 으로 각각 필요한 컨트롤러들을 만들어준다.
경로도 제대로 설정해주고 `path, pagetitle` 도 같이 보내준다.
이렇게하면 기본적인 컨트롤러 및 라우트 설정이 거의 끝난다.

이제 `admin.js` 에 가면 `products` 라우트가 있는데, 컨트롤러 폴더의 `admin.js` 파일에 가서 새로운 라우트를 추가한다.
`getProducts` 로 일반 미들웨어 함수를 만들어서, 모든 제품을 가져와서 뷰를 렌더링하려고한다.

```js
// admin.js
const Product = require("../models/product");

exports.getAddProduct = (req, res, next) => {
  res.render("admin/add-product", {
    pageTitle: "Add Product",
    path: "/admin/add-product",
    formsCSS: true,
    productCSS: true,
    activeAddProduct: true,
  });
};

exports.postAddProduct = (req, res, next) => {
  const product = new Product(req.body.title);
  product.save();
  res.redirect("/");
};

exports.getProducts = (req, res, next) => {
  Product.fetchAll((products) => {
    res.render("admin/products", {
      prods: products,
      pageTitle: "Admin Products",
      path: "/admin/products",
    });
  });
};
```

이렇게 설정을 완료하고 새로고침하면 `흰 페이지` 가 보이는데 일단 오류는 아니기 대문에 문제는 없다. 현재 접속중인 라우트 / 뒤에 아무것도 없어서 `shop` 컨트롤러에서 `getIndex` 를 실행하게 되어있는데 이는 `shop/index` 이고 텅빈 `ejs` 뷰 이기 때문이다.
따라서 빈 페이지가 표시되는 것이 맞음.

`product-list.ejs` 의 코드를 `index.ejs` 에 복사해 넣어주면 제대로 뷰가 나오는 것을 알 수 있다.

이후 `Admin Products` 를 클릭하면
![](https://velog.velcdn.com/images/chromeheartz/post/aa28d609-aa40-4ca1-89a0-8fe53a60d108/image.png)
이렇게 오류가 나는데 이는 2가지를 의미한다.

- 라우트가 잘못되었음.
  `routes` 폴더의 admin.js에서는 `/products` 가 맞다.
  admin 컨트롤러를 연결하지 않았기 때문에 `routes/admin.js` 에 가서 연결해준다.
  이후 경로가 정의되지 않았다고 하는데, `404 controller` 정확히는 오류 컨트롤러에 경로를 추가해야 한다.

```js
// error.js
exports.get404 = (req, res, next) => {
  res.status(404).render("404", { pageTitle: "Page Not Found", path: "/404" });
};
```

이제 새로고침하면 제대로 동작한다.
