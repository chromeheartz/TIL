### 📌 제품 모델

이제 모델에 대해 생각해볼 시간이다.
일단 현재 모델이 지니는 문제점은 매우 단순하다는 것이다. 제품 배열을 관리하게 되는데 제품은 `{title: req.body.title}` 형태일것이다.
나중에는 유저를 비롯한 다른 요소들도 구비하게 되지만, 일단 제품에만 집중해보자.

`models` 라는 폴더를 만들어놓자. 그렇게 되면 `MVC` 패턴이 완성이 된다.
여기에 `product.js` 파일을 추가한다. 단수형의 이름인것에 주의하자. `최종 코어 데이터` 는 하나의 제품이므로 `단일 엔티티` 를 나타내려고 한다.

단일 제품이야 말로 앱을 최종적 혹은 부분적으로 정의하는 요소이다

#### 그럼 모델은 어떤 형태를 지닐까?

```js
// models/product.js
const products = [];

module.exports = class Product {
  constructor(t) {
    this.title = t;
  }

  save() {
    products.push(this);
  }

  static fetchAll() {
    return products;
  }
};
```

`class` 를 생성해서 생성자를 만들 수 있게 해주자.
여기에서는 제품에 대한 제목을 수신하고, 컨트롤러 내부에서 생성하려고 한다. 그러니 제목을 가져오고 이때 이름도 지정해줄 수 있다.
클래스에 속성을 생성하는데, `this` 키워드로하고 `this.title` 을 인수로 수신하는 `title` 과 동일하게 설정해준다. 이름들은 정확히 일치하게 할 필요는 없고, 일단 `t` 로 이름을 짓고 진행해보자.

이 클래스에 속성을 생성하고 있는데, 이후에는 클래스를 기반으로 객체를 생성함으로써 `New` 로 호출하게 될 구축자로 title을 전달할 수 있게 된다.

하지만 단순히 title이 있는 객체를 생성하는 것에서 그치는 것은 아니다.
제품 배열을 여기에 만들어놓고 저장하고 가져올 수 있도록 해보자.
`save` 메소드를 만들어서 `products.push(this)` 로 추가해준다. 클래스를 기반으로 생성된 객체를 참조할것인데 이 배열에 저장하고자 하는 그 객체가 해당되기 때문이다.

`fetchAll` 이라는 함수를 만들어서 모든 제품을 가져오도록 해보자. 제품의 `단일 인스턴스` 에는 호출되지 않는다. 그리고 모든 존재하는 제품을 가져오기 위해 `static` 키워드를 사용할것인데, 이걸 사용하면 예시된 객체가 아니라 `클래스 자체` 에 이 방법을 호출할 수 있게 된다.

이렇게 되면 `controllers/products.js` 에서도 `const Product = require('../models/product')` 로 가져와서 써야 한다.
