### 04. 모델을 통해 파일에 데이터 저장 & 가져오기

### 📌 모델을 통해 파일에 데이터 저장하기

`save` 를 호출할 때 파일에 저장하게 된다. 그리고 해당 파일에는 옛날 제품과 새로운 제품 모두 저장해야 한다.
먼저 파일시스템을 작업해야 하니 `fs module` 에서 `fs` 를 임포트한다.
save에서 path모듈을 사용하여 `모든 운영체제` 에서 작동하는 경로를 구축해야 한다.

```js
// util/path.js
const path = require("path");

module.exports = path.dirname(process.mainModule.filename);
```

```js
const p = path.join(
  path.dirname(process.mainModule.filename),
  "data",
  "products.json",
);
```

`path.dirname(process.mainModule.fileName)` 를 사용한다.
하지만 단지 이것은 `루트 디렉토리` 이고 이 안에 새로 데이터 폴더를 미리 생성함으로써 `권한 문제` 를 방지하도록 하자.
`data/product.json` 파일을 추가해준다.

```js
fs.readFile(p, (err, fileContent) => {
  let products = [];
  if (!err) {
    products = JSON.parse(fileContent);
  }
  products.push(this);
  fs.writeFile(p, JSON.stringify(products), (err) => {
    console.log(err);
  });
});
```

먼저 기존 파일을 읽기 위해 `fs.readFile` 을 사용해서 파일의 전체 콘텐츠를 읽어온다. 참고로 파일 크기가 큰 경우 더 효율적인 방법이 있는데, 작업하기에 앞서 전부 메모리로 읽어올 필요는 없기 때문이다.
`Stream` 이라는 함수를 읽을 수도 있다. `createReadStream` 함수를 사용가능.
하지만 현재는 일단 전체 파일을 읽는 형태로 진행.
이 경로에 있는 파일이자 지금 우리가 관심있는 `p` 파일을 읽어서 다음 작업을 진행한다.
오류나 데이터 파일 콘텐츠를 가져오도록 할것인데 `버퍼` 가 좀 발생할것이다.

초기에는 공백 배열을 만들어 놓고 오류가 있을 경우에는 공백으로 남겨두자.
하지만 오류가 없을 경우 즉, `null` 이라면 추출했던 파일로부터 제품들을 읽어오면 된다.

추출했던 파일로부터 제품들을 읽어올것인데, 따라서 `fileContent` 에 대응하는 값이 존재할테고, 콘텐츠에 해당할텐데 그 파일이 `JSON` 파일이므로 `JSON 형식` 으로 저장한다.

그럼 `products = JSON` 으로 설정하는데 Node.js에 존재하는 핼퍼 객체이므로 직접 정의할 필요가 없다. 들어오는 JSON을 받아 JS배열, 객체 또는 파일에 있는 내용을 돌려주는 `parse` 메소드를 사용.
이후에 `this` 에 해당하는 객체를 추가해준다.

> ✅ 중요한 점은 `this` 가 클래스를 참조하도록 하기 위해 화살표 함수를 사용해주어야 하는데 그렇지 않으면 문맥을 잃어버리고 더 이상 클래스를 참조하지 않기 때문이다

남아있는 작업은 `파일로 다시 저장` 해주어야 하는 것이다.
이를 위해 `writeFile` 을 사용하여 읽은 경로와 동일한 경로에 작성하고 `JSON 데이터` 를 배치할것이다.`JSON` 으로 변환하여 올바른 형식을 지니도록 하는 `Stringify` 메소드르 사용할 수 있다.

오류가 발생하는 경우를 대비해 콜백을 입력해준다.

```js
static fetchAll(cb) {
  const p = path.join(
    path.dirname(process.mainModule.filename),
    'data',
    'products.json'
  );
  // 첫번째
  fs.readFile(p, (err, fileContent) => {
  	if (err) {
    	return [];
    }
  });
  // 두번째
  fs.readFile(p, (err, fileContent) => {
    if (err) {
      cb([]);
    }
    cb(JSON.parse(fileContent));
  });
}
```

```js
// controller/products.js
exports.getProducts = (req, res, next) => {
  Product.fetchAll((products) => {
    res.render("shop", {
      prods: products,
      pageTitle: "Shop",
      path: "/",
      hasProducts: products.length > 0,
      activeShop: true,
      productCSS: true,
    });
  });
};
```

`fetchAll` 같은 경우도 파일을 읽어오도록 해야 한다.
오류, 파일 콘텐츠도 있고, 사용하고 싶은 데이터를 붙여준다.

첫번째 코드로 하게되면 제품들에 length가 없다는 오류가 계속 나타나고 있다.
이건 `비동기식 코드` 라는 것을 유념해야 한다. 따라서 `fetchAll` 메소드는 실행하는데 콜백을 `이벤트 이미터 레지스트리` 에 등록하여 배치하게 된다. 하지만 그 당므에는 함수를 그냥 끝내버리고 이 함수는 아무것도 반환하지 않는다.

`두 번째` 방식으로 고칠 수 있다.
✅ 단순히 `fetchAll` 에 콜백인수를 포함시키면 된다. 이를 통해 `fetchAll` 에 함수를 전달할 수 있게 되고, 함수가 끝난 뒤 `fetchAll` 이 함수를 실행한다.

> 함수를 전달할 수 있고 이후 반환하려는 데이터의 호출을 인식한다.

함수를 수신하게 될 텐데 이 인수는 함수를 포착하게 되고 (cb) 배열을 반환하는 대신 공백 배열을 전달할 콜백이 들어오게 된다.아래에서도 동일하게 진행하되 공백 배열은 없고 대신 `파싱된 JSON 데이터` 를 전달하는 콜백이 들어온다.

`controller` 에서도 콜백 과정을 생성하고 제품을 모두 가져오는 과정이 `끝난 뒤에` fetchAll에 전달하게 될 함수를 렌더링 하게 만들어준다.

```js
// models/product.js
const fs = require("fs");
const path = require("path");

module.exports = class Product {
  constructor(t) {
    this.title = t;
  }

  save() {
    const p = path.join(
      path.dirname(process.mainModule.filename),
      "data",
      "products.json",
    );
    fs.readFile(p, (err, fileContent) => {
      let products = [];
      if (!err) {
        products = JSON.parse(fileContent);
      }
      products.push(this);
      fs.writeFile(p, JSON.stringify(products), (err) => {
        console.log(err);
      });
    });
  }

  static fetchAll(cb) {
    const p = path.join(
      path.dirname(process.mainModule.filename),
      "data",
      "products.json",
    );
    fs.readFile(p, (err, fileContent) => {
      if (err) {
        cb([]);
      }
      cb(JSON.parse(fileContent));
    });
  }
};
```
