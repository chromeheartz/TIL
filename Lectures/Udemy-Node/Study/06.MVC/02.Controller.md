### 📌 Controller

아직 `router` 함수에 컨트롤러들이 뒤섞여있다.
라우팅하는 방법은 바뀌지 않는데, `router.get` 같이 사용하는 방식은 달라지지 않는다.
하지만 실행된 논리는 사실 `전형적인 컨트롤러 논리이다`

한줄 뿐이기는 하지만 데이터와 교류하고 있고, 뷰를 반환하는 것이 정확히 컨트롤러를 구성하는 `중간 논리(In-between Logic)` 이다.
모든 파일을 `routes` 파일에 넣으면 아주 큰 파일이 되기 때문에 여러 파일에 나눠 넣는것이 좋다.
일단 `controllers` 라는 폴더를 만들어서 `products` 들에 대한 컨트롤러를 나누어보자.

```js
// controllers/products.js
const products = [];

exports.getAddProduct = (req, res, next) => {
  res.render("add-product", {
    pageTitle: "Add Product",
    path: "/admin/add-product",
    formsCSS: true,
    productCSS: true,
    activeAddProduct: true,
  });
};

exports.postAddProduct = (req, res, next) => {
  products.push({ title: req.body.title });
  res.redirect("/");
};

exports.getProducts = (req, res, next) => {
  res.render("shop", {
    prods: products,
    pageTitle: "Shop",
    path: "/",
    hasProducts: products.length > 0,
    activeShop: true,
    productCSS: true,
  });
};

// admin.js
const path = require("path");
const express = require("express");
const productsController = require("../controllers/products");
const router = express.Router();

// /admin/add-product => GET
router.get("/add-product", productsController.getAddProduct);
// /admin/add-product => POST
router.post("/add-product", productsController.postAddProduct);

module.exports = router;
```

이렇게 정리하면 보기 깔끔하다. `getProducts` 처럼 http메소드를 앞에 붙여놓으면 조금더 직관적으로 알기 편하다. `exports.` 로 한 파일에서 여러 내보내기를 해준다
함수들을 보면 `Express` 가 이해하는 일반적인 미들웨어 함수이기 때문에 `req, res, next` 를 받는다. 이렇게 컨트롤러를 만들고 나면 `admin.js` 에서 `router.get` 으로 가져와서 컨트롤러 함수를 넣어주기만 하면 된다.

다시 말해 `Express` 라우터에 이 함수를 저장하라고 알려주는 것이다. 라우트에 요청이 도달할때마다 이 함수를 실행하게 될것이다.
또한 현재 `products` 가 `admin.js` 에 없으니 `module.exports = products` 로 내보내줄 필요 또한 없어진다. 대신 `shop.js` 파일에서 내보내기를 해야한다.

```js
// shop.js
const path = require("path");
const express = require("express");
const productsController = require("../controllers/products");
const router = express.Router();

router.get("/", productsController.getProducts);

module.exports = router;
```

같은 맥락으로 `shop.js` 또한 수정해준다.

이제 `404` 라우트를 처리할 수 있도록 만드는데 이론상으로 꼭 필요한 것은 아니다.

```js
exports.get404 = (req, res, next) => {
  res.status(404).render("404", { pageTitle: "Page Not Found" });
};
```
