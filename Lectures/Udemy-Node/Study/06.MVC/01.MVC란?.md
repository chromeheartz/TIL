### 📌 MVC란?

![](https://velog.velcdn.com/images/chromeheartz/post/452f1a08-b9f7-48ec-b6fa-0ac2117487fe/image.png)

`MVC` 는 무슨 약자이며 어던 역할을 할까

> ✅ 쉽게 말하면 관심사를 분리하는 것이다.
> 코드의 여러 부분들이 각기 다른 기능을 가져서 어떤 부분이 어떤 작업을 책임지고 있는지 확실하게 아는것이다.

`MVC` 는 `Model View Controller` 의 약자로 `모델, 뷰, 컨트롤러` 와 관련되어있는데, 그 중 `view` 는 이미 `pug, handlebars, ejs` 에서 확인한 적이 있다.

모델은 기본적으로 `객체 / 데이터` 를 나타내는 코드의 한 부분으로 데이터를 저장하거나 파일로부터 데이터를 주고받는 등, 데이터 관련 작업을 할 수 있도록 한다.

현재 하던 것처럼 메모리에만 있어도 `Models` 에 의해 처리된다. `Views` 는 사용자가 보게 되는 화면을 책임지고, html자료에 올바른 내용을 렌더링해 사용자에게 보내는 역할을 한다.
애플리케이션 코드와 분리되어 있으며 뷰를 생성하기 위해 템플릿 엔진에 주입하는 데이터와 아주 약간만 통합되어 있다.

> `Controllers` 는 `Models / Views` 사이의 연결점이다.
> 뷰는 애플리케이션 논리와 상관이 없으며 모델은 데이터를 저장하고 가져오는 역할을 하기 때문에 컨트롤러가 모델과 함께 데이터를 저장하거나 저장 프로세스를 유발한다.

뷰에서 가져온 데이터를 전달하는 경우도 마찬가지로 컨트롤러가 돕게 된다.
즉, 컨트롤러는 중개인으로서 중간 논리를 포함한다.

#### 라우트는 이들과 어떤 관계일까?

`라우트` 는 어떤 경로에 따른 `http 메소드` 에 따라 어떤 컨트롤러 코드를 실행할지 정의한다.
컨트롤러는 작업할 모델과 렌더링할 뷰를 정의하는 역할을 하는데, 이것이 패턴이며 `Express` 가 내장된 앱은 미들웨어 개념에 크게 기반해있다.

컨트롤러는 미들웨어 기능간에 분할되는데, 다시 말해 일부 논리가 분리되어 다른 미들웨어 기능으로 이동하는 것이다.
