### 📌 react live source code update

이제 프론트엔드에서도 `live source code update` 를 해보자.
소스코드에서 무언가를 변경하면 실행 중인 컨테이너에 자동으로 반영되도록. 일단 컨테이너를 종료하고 `-it` 옵션을 사용해 인터렉티브 액세스를 사용했기 때문에 `cmd + c` 로도 컨테이너를 종료시킬 수 있다.

이제 `바인드 마운트` 를 필요로 하는데, 데이터 지속성같은 것은 필요 없다.
그래서 `docker run` 으로 컨테이너를 실행하지만 `-v` 를 설정해서 바인딩을 할것인데, `/app` 폴더에 바인딩해보자. 전체 폴더를 바인딩할 필요는 없고 `/src -> /app/src` 으로 바인딩하면 된다. `/src` 소스 폴더가 일반적으로 작업하는 react 코드가 있는 곳이기 때문.
로컬 호스트 머신 경로에 바인딩 할것인데,

`docker run -v /Users/bibi/Documents/GitHub/TIL/Lectures/Udemy-Docker/Code/05-multi-starting-setup/frontend/src:/app/src --name goals-frontend --network goals-net --rm -p 3000:3000 -it goal-react`

이런식으로 작성할것이다. 백엔드에서 사용한 `nodemon` 이 필요하지 않은 이유는 `react` 개발 서버는 이미 변경사항을 적용하고 앱을 다시 빌드하도록 구성되어 있기 때문에 부가적인 종속성이 필요하지 않다.
