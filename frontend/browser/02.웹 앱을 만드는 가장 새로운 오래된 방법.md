# htmx: 웹 앱을 만드는 가장 새로운 오래된 방법

> 원문: https://blog.logrocket.com/htmx-the-newest-old-way-to-make-web-apps/
> 번역 출처: https://velog.io/@eunbinn/htmx-the-newest-old-way-to-make-web-apps

![](https://velog.velcdn.com/images/chromeheartz/post/476509ba-a005-4e2c-8817-31b1363dbfde/image.png)

요즘 프론트엔드 생태계는 React, Vue, Svelte 같은 JS 프레임워크가 지배하고 있는데, htmx는 완전히 반대 방향을 택한다. "그냥 HTML 속성으로 다 해버리자"는 발상인데, 생각보다 꽤 쓸 만하다.

### htmx가 뭔데

HTML 요소에서 직접 AJAX 요청 보내고, CSS 트랜지션 트리거하고, WebSocket/SSE 연결까지 할 수 있는 자바스크립트 라이브러리다. 단, JS를 쓰는 게 아니라 **HTML 속성**으로 모든 걸 선언한다는 게 핵심.

스펙이 꽤 가볍다:

- **약 10KB** (min+gzip)
- 의존성 없음
- IE11 호환

### 설치

CDN으로 한 줄이면 끝.

```html
<script src="https://unpkg.com/htmx.org@1.3.3"></script>
```

### AJAX 요청 보내기

htmx가 지원하는 속성들:

| 속성        | HTTP 메서드 |
| ----------- | ----------- |
| `hx-get`    | GET         |
| `hx-post`   | POST        |
| `hx-put`    | PUT         |
| `hx-patch`  | PATCH       |
| `hx-delete` | DELETE      |

가장 단순한 예:

```html
<!-- 버튼 클릭 시 GET 요청 -->
<button hx-get="http://localhost/todos">Load Todos</button>
```

버튼을 클릭하면 해당 URL로 GET 요청을 보내고, 서버에서 받은 HTML을 버튼 안에 그냥 넣어버린다.

### 트리거 — 언제 요청을 보낼지

기본 트리거는 요소 타입에 따라 자동으로 결정된다:

- `input`, `select`, `textarea` → `change` 이벤트
- `form` → `submit` 이벤트
- 그 외 → `click` 이벤트

`hx-trigger`로 커스터마이징 가능:

```html
<!-- mouseenter 시 요청 -->
<div hx-get="http://localhost/todos" hx-trigger="mouseenter">
  Mouse over me!
</div>
```

#### 트리거 수정자

트리거에 수정자를 붙여서 세밀하게 제어할 수 있다:

| 수정자            | 설명                        |
| ----------------- | --------------------------- |
| `once`            | 딱 한 번만 실행             |
| `changed`         | 값이 바뀔 때만 요청         |
| `delay:<time>`    | n초 후 실행 (디바운스 느낌) |
| `throttle:<time>` | n초에 한 번만 허용          |
| `from:<selector>` | 다른 요소의 이벤트를 감지   |

실전에서 검색창에 자주 쓰게 될 조합:

```html
<!-- 키를 뗄 때, 값이 바뀐 경우에만, 500ms 디바운스 -->
<input
  type="text"
  hx-get="http://localhost/search"
  hx-trigger="keyup changed delay:500ms"
/>
```

#### 폴링

일정 주기로 서버에 요청 보내기도 된다. SSE 붙이기 귀찮을 때 간이 방법으로 쓸 수 있을 것 같다.

```html
<!-- 2초마다 GET 요청 -->
<div hx-get="/history" hx-trigger="every 2s"></div>
```

### 응답 HTML을 어디에, 어떻게 넣을지

#### `hx-target` — 대상 지정

기본적으로 요청을 보낸 요소 자체에 응답이 들어가는데, `hx-target`으로 다른 요소에 넣을 수 있다.

```html
<button hx-get="http://localhost/todos" hx-target="#result">Load Todos</button>
<div id="result"></div>
```

#### `hx-swap` — 삽입 방식

| 값            | 동작                        |
| ------------- | --------------------------- |
| `innerHTML`   | 기본값. 요소 내부를 교체    |
| `outerHTML`   | 요소 자체를 교체            |
| `afterbegin`  | 첫 번째 자식 앞에 삽입      |
| `beforebegin` | 요소 앞(부모 기준)에 삽입   |
| `beforeend`   | 마지막 자식 뒤에 삽입       |
| `afterend`    | 요소 다음(부모 기준)에 삽입 |
| `none`        | 응답을 DOM에 반영하지 않음  |

### 로딩 인디케이터

`htmx-indicator` 클래스를 붙이면 AJAX 요청 중에만 보이는 요소가 된다. 기본 opacity가 0이고, htmx가 요청 시 `htmx-request` 클래스를 부모에 추가해서 opacity를 1로 만든다.

```html
<div hx-get="http://path/to/api">
  <button>Click Me!</button>
  <img class="htmx-indicator" src="path/to/spinner.gif" />
</div>
```

### 데이터 전송

#### 기본 동작

폼이나 입력 요소가 트리거가 되면 모든 입력값이 자동으로 요청에 포함된다.

#### `hx-include` — 원하는 요소만 골라서 포함

```html
<div>
  <button hx-post="http://path/to/api/register" hx-include="[name=username]">
    Register!
  </button>
  Enter Username: <input name="username" type="text" />
</div>
```

#### `hx-params` — 파라미터 필터링

| 값           | 동작                 |
| ------------ | -------------------- |
| `*`          | 모든 파라미터 포함   |
| `none`       | 파라미터 없이 요청   |
| `not <list>` | 특정 파라미터 제외   |
| `<list>`     | 특정 파라미터만 포함 |

#### 파일 업로드

`hx-encoding="multipart/form-data"`를 폼에 붙여줘야 파일 업로드가 된다.

```html
<form hx-encoding="multipart/form-data">
  Select File:
  <input type="file" name="myFile" />
  <button hx-post="http://path/to/api/register" hx-include="[name='myFile']">
    Upload File!
  </button>
</form>
```

### 유효성 검사

htmx는 HTML5 기본 유효성 검사 API와 자동으로 통합된다. 폼 입력이 invalid 상태면 요청 자체를 보내지 않는다.

커스텀 로직이 필요하면 이벤트를 활용:

- `htmx:validation:validate` — 검사 직전에 호출
- `htmx:validation:failed` — 검사 실패 시
- `htmx:validation:halted` — 에러로 인해 요청이 막혔을 때

### CSS 애니메이션 — `class-tool`

자바스크립트 없이 클래스를 추가/제거/토글할 수 있다. `classes` 속성으로 선언.

```html
<!-- 1초 후 sample-class 추가 -->
<div classes="add sample-class:1s"></div>
```

**순차 실행** (쉼표로 구분):

```html
<div
  classes="add sample-class:1s, remove another-class:2s, toggle 3rd-class:1s"
></div>
```

**동시 실행** (`&`로 구분):

```html
<div
  classes="add sample-class:1s & remove another-class:2s & toggle 3rd-class:1s"
></div>
```

실제 페이드 인/아웃 예제:

```html
<style>
  .demo.faded {
    opacity: 0.3;
  }
  .demo {
    opacity: 1;
    transition: opacity ease-in 900ms; /* CSS transition이 실제 애니메이션을 담당 */
  }
</style>
<div class="demo" classes="toggle faded:1s">I'm Fading! ⚡</div>
```

htmx는 클래스만 토글해주고, 실제 부드러운 움직임은 CSS `transition`이 처리한다. 역할 분리가 깔끔하다.

### 📌 정리

`"JS 없이 HTML 속성만으로 동적인 웹앱을 만든다" — 이게 htmx의 전부다`

#### 핵심

> htmx는 HTML을 하이퍼미디어의 원래 의도대로 확장한다. 브라우저가 원래 할 수 있어야 했던 것들(AJAX, 부분 업데이트, 이벤트 기반 상호작용)을 JS 없이 HTML 속성으로 선언한다. React 같은 프레임워크가 "JS 안에서 HTML을 쓰는" 방향이라면, htmx는 "HTML 안에서 로직을 쓰는" 방향이다.

#### ⭐️ 앞으로 기억할 것

- 1. **hx-trigger 수정자 조합이 실용적이다**
     검색창 디바운스 같은 흔한 패턴을 속성 하나로 해결할 수 있다.

  ```html
  <input hx-get="/search" hx-trigger="keyup changed delay:500ms" />
  ```

- 2. **서버가 HTML을 응답해야 한다**
     JSON이 아니라 HTML 조각(fragment)을 응답으로 내려줘야 한다. htmx는 그걸 받아서 DOM에 그냥 꽂는다. 서버 사이드 렌더링 친화적인 구조.

- 3. **hx-swap으로 삽입 위치를 세밀하게 제어**
     `beforeend`를 쓰면 무한 스크롤처럼 목록 끝에 계속 추가하는 패턴이 쉽다.

  ```html
  <div hx-get="/more-items" hx-swap="beforeend" hx-target="#list">더 보기</div>
  <ul id="list">
    ...
  </ul>
  ```

- 4. **WebSocket/SSE도 지원 (실험적)**
     실시간 기능도 htmx 속성으로 연결할 수 있다. 아직 실험적이라 프로덕션엔 신중하게.

📌 결론
▎ htmx는 SPA가 과한 경우 — 서버 사이드 렌더링 기반 앱에 동적인 맛만 살짝 추가할 때 — 딱 맞는 선택이다.
▎ 프레임워크 번들 없이 10KB로 AJAX, 폴링, 애니메이션, 파일 업로드를 다 처리한다.
▎ ⭐️ 실무 적용 포인트: 관리자 페이지, 인트라넷, MPA 구조의 프로젝트에서 "이 버튼 하나만 비동기로 만들고 싶다"는 상황에 htmx가 React보다 훨씬 가볍고 빠른 해법이 된다.
