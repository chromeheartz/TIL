
# 리액트 컴파일러

> **원문**: https://react.dev/learn/react-compiler
> **번역 출처**: https://velog.io/@eunbinn/react-compiler

![](https://velog.velcdn.com/images/chromeheartz/post/56151077-7334-42f6-b2f1-b7d688395654/image.png)

React 공식 문서에서 컴파일러 소개 글이 올라왔길래 읽어봤다. `useMemo`, `useCallback`, `React.memo`를 직접 쓰지 않아도 컴파일러가 자동으로 메모화해준다는 내용인데, 바로 도입하기보다 어떻게 동작하는지 파악해두는 게 목적이었음.

---

### 리액트 컴파일러가 하는 일

한 마디로 — **자동 메모화(auto-memoization)**다.

지금까지 성능 최적화를 위해 수동으로 해야 했던 것들:
- `useMemo` — 복잡한 계산 캐싱
- `useCallback` — 함수 참조 유지
- `React.memo` — 자식 컴포넌트 리렌더링 방지

컴파일러가 이걸 빌드 타임에 자동으로 처리한다.

#### 두 가지 최적화 유형

**1. 폭포식 리렌더링 차단**
부모가 바뀌어도 props가 변경되지 않은 자식 컴포넌트는 리렌더링하지 않는다.

```javascript
function FriendList({ friends }) {
  const onlineCount = useFriendOnlineCount();
  if (friends.length === 0) {
    return <NoFriends />;
  }
  return (
    <div>
      <span>{onlineCount} online</span>
      {friends.map((friend) => (
        <FriendListCard key={friend.id} friend={friend} />
      ))}
      <MessageButton />
    </div>
  );
}
```

컴파일러가 `FriendListCard`, `MessageButton` 등을 자동으로 메모화한다.

**2. 비싼 계산 자동 캐싱**

```javascript
function TableContainer({ items }) {
  const data = expensivelyProcessAReallyLargeArrayOfObjects(items);
  // 컴파일러가 items가 바뀌지 않으면 재계산하지 않도록 처리
}
```

---

### 컴파일러가 가정하는 것들

컴파일러가 안전하게 최적화하려면 세 가지 조건을 충족해야 한다:

1. 유효한 시맨틱 자바스크립트 코드
2. nullable/optional 값 접근 전 체크 (TypeScript `strictNullChecks` 켜는 걸 권장)
3. **리액트의 규칙 준수** — 훅 규칙, 렌더링 순수성 등

규칙을 위반하는 컴포넌트는 컴파일러가 최적화를 건너뛰고, 오류를 내지 않는다. 안전하게 패스하는 것.

### 시작하기

#### 사전 호환성 체크

```bash
npx react-compiler-healthcheck@latest
```

```
Successfully compiled 8 out of 9 components.
StrictMode usage not found.
Found no usage of incompatible libraries.
```

컴파일 가능한 컴포넌트 비율, StrictMode 사용 여부, 호환되지 않는 라이브러리 여부를 알려준다.

#### ESLint 플러그인 (컴파일러 없이도 유용)

컴파일러와 별개로 리액트 규칙 위반을 잡아주는 플러그인이다. 컴파일러 도입 전에 먼저 설치해두면 좋다.

```bash
npm install eslint-plugin-react-compiler
```

```javascript
module.exports = {
  plugins: ["eslint-plugin-react-compiler"],
  rules: {
    "react-compiler/react-compiler": "error",
  },
};
```

### 프레임워크별 설치

#### Vite

```bash
npm install babel-plugin-react-compiler
```

```javascript
// vite.config.js
export default defineConfig(() => ({
  plugins: [
    react({
      babel: {
        plugins: [["babel-plugin-react-compiler", ReactCompilerConfig]],
      },
    }),
  ],
}));
```

#### Next.js

```bash
npm install next@canary babel-plugin-react-compiler
```

```javascript
// next.config.js
module.exports = {
  experimental: {
    reactCompiler: true,
  },
};
```

App Router, Pages Router, Webpack, Turbo 모두 지원한다.

#### 기존 프로젝트 — 점진적 도입

전체 코드베이스에 한 번에 적용하지 말고 특정 디렉토리부터 시작:

```javascript
const ReactCompilerConfig = {
  sources: (filename) => {
    return filename.indexOf("src/path/to/dir") !== -1;
  },
};
```

또는 `"use memo"` 어노테이션으로 컴포넌트 단위로 opt-in:

```javascript
export default function App() {
  "use memo";
  // 이 컴포넌트만 컴파일러 적용
}
```

### 문제 해결

#### 컴파일 후 컴포넌트가 오작동할 때

`"use no memo"`로 해당 컴포넌트를 일시적으로 제외:

```javascript
function SuspiciousComponent() {
  "use no memo"; // 임시 탈출구 — 수정 후 반드시 제거할 것
}
```

ESLint 플러그인 경고를 먼저 확인하고, 리액트 규칙 위반이 있는지 체크하는 게 우선이다.

#### 최적화 확인

리액트 개발자 도구(v5.0 이상)에서 컴파일러가 최적화한 컴포넌트에 **"메모 ✨"** 배지가 붙는다.

#### `(0 , _c) is not a function` 에러

리액트 19 RC 이상으로 업그레이드 필요.

### 📌 정리

`수동 useMemo/useCallback 없이 컴파일러가 자동으로 메모화한다`

#### 핵심
> 리액트 컴파일러는 리액트 규칙을 지키는 코드를 빌드 타임에 분석해서 자동으로 메모화를 삽입한다.
> 규칙을 지키지 않는 컴포넌트는 최적화를 건너뛰기 때문에, 도입 전에 ESLint 플러그인으로 규칙 위반을 먼저 정리하는 게 순서다.

#### ⭐️ 앞으로 기억할 것

- 1. **도입 순서: ESLint 플러그인 → healthcheck → 컴파일러**
컴파일러보다 ESLint 플러그인이 먼저다. 규칙 위반 잡고, healthcheck로 호환성 확인하고, 그다음 컴파일러 도입.
```bash
npx react-compiler-healthcheck@latest
```

- 2. **점진적 도입 — 특정 디렉토리나 `"use memo"` 어노테이션으로**
기존 프로젝트에 전체 적용은 리스크가 있다. 먼저 작은 범위에서 검증하기.

- 3. **`"use no memo"`는 임시 탈출구**
오작동 컴포넌트를 제외할 때 쓰지만, 해결 후 반드시 제거해야 한다. 안 지우면 영구적으로 최적화가 건너뛰어진다.

- 4. **아직 안정화 전 — 새 프로젝트에만 적극 고려**
기존 프로젝트는 안정 릴리스 이후 도입해도 늦지 않다. 지금은 동작 원리 파악이 목표.

> **📌 결론**
> ▎ 컴파일러 = useMemo/useCallback/React.memo를 자동으로 삽입해주는 빌드 도구
> ▎ 리액트 규칙을 잘 지킨 코드일수록 컴파일러가 더 많이 최적화한다
> ▎ ⭐️ 지금 당장 쓸 게 아니라도 ESLint 플러그인은 바로 도입할 수 있고 그것만으로도 가치 있다
