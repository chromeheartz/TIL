# 리액트 동시성 이해하기

> 원문: https://www.bbss.dev/posts/react-concurrency/
> 번역 출처: https://velog.io/@eunbinn/react-concurrency

![](https://velog.velcdn.com/images/chromeheartz/post/da614fc2-4128-4a08-96bd-d5488ef7bb61/image.png)

React 18에서 `useTransition`, `useDeferredValue` 같은 훅이 생겼는데, "동시성"이라는 말만 보면 뭔가 거창해 보여서 미뤄뒀었다. 근데 정리를 해보니 생각보다 개념이 명확하다.

### 동시성 렌더링이 뭔데

한 줄 요약: **다음 뷰를 렌더링하는 동안에도 현재 뷰가 반응하게 만드는 것**이다.

기존 React 렌더링은 블로킹 방식이다. 컴포넌트 트리를 위에서 아래로 순서대로, 끊김 없이 다 돌린다.

```js
// 블로킹 렌더링 — 한 번 시작하면 다 끝날 때까지 멈추지 않는다
function renderBlocking(Component) {
  for (let Child of Component) {
    renderBlocking(Child);
  }
}
```

무거운 컴포넌트가 있으면? 렌더링이 끝날 때까지 UI가 굳어버린다. 버튼 클릭해도 반응 없고, 입력창에 타이핑해도 버벅이는 그 느낌.

동시성 렌더링은 이걸 쪼갠다. `requestIdleCallback`처럼 브라우저가 한가할 때 조금씩 처리하고, 더 중요한 작업이 생기면 멈출 수 있다.

```js
// 동시성 렌더링 — 중간에 취소 가능, 브라우저 한가할 때 조금씩
function renderConcurrent(Component) {
  if (isCancelled) return; // 중간에 취소 가능

  for (let Child of Component) {
    requestIdleCallback(() => renderConcurrent(Child));
  }
}
```

### Concurrent Mode가 아니라 Concurrent Features

원래 계획은 "Concurrent Mode"라는 걸 만들어서 통째로 전환하려 했는데, 하위 호환성 문제로 포기했다. 대신 기존 코드를 건드리지 않고 **선택적으로 쓸 수 있는 훅** 두 개를 내놨다.

### `useTransition`

```js
const [isPending, startTransition] = useTransition();
```

- `startTransition(fn)` — 여기 안에 넣은 상태 업데이트는 "급하지 않은 것"으로 표시된다
- `isPending` — 동시 렌더링이 진행 중이면 `true`

```js
function MyCounter() {
  const [isPending, startTransition] = useTransition();
  const [count, setCount] = useState(0);

  const increment = useCallback(() => {
    startTransition(() => {
      setCount((c) => c + 1); // 이 업데이트는 낮은 우선순위로 처리
    });
  }, []);

  return (
    <>
      <button onClick={increment}>Count {count}</button>
      <span>{isPending ? "Pending..." : "Done"}</span>
      <ManySlowComponents count={count} /> {/* 무거운 컴포넌트 */}
    </>
  );
}
```

내부 동작을 단순화하면 이렇다:

```js
function startTransition(stateUpdates) {
  isInsideTransition = true;
  stateUpdates(); // 상태 업데이트 실행
  isInsideTransition = false;
}

function setState() {
  if (isInsideTransition) {
    // 동시 렌더링 예약 (낮은 우선순위)
  } else {
    // 블로킹 렌더링 예약 (높은 우선순위)
  }
}
```

**제어된 입력(controlled input)에는 쓰면 안 된다.** `<input value={state} onChange={...}>`처럼 입력값이 state에 묶여 있는 경우, `startTransition`으로 낮은 우선순위로 처리하면 타이핑할 때 입력창이 버벅인다. 이 경우엔 `useDeferredValue`를 써야 한다.

### `useDeferredValue`

상태 업데이트를 `startTransition`으로 감쌀 수 없는 상황 — 예를 들어 외부 라이브러리에서 값이 오거나, 부모 컴포넌트에서 props로 내려오는 경우 — 에 쓴다.

개념적으로는 디바운스랑 비슷하다. "이 값의 업데이트는 바쁠 때 잠깐 미뤄도 돼"라는 표시.

내부적으로는 이렇게 동작한다:

```js
function useDeferredValue(value) {
  const [isPending, startTransition] = useTransition();
  const [state, setState] = useState(value);

  useEffect(() => {
    startTransition(() => {
      setState(value); // 값 변경을 낮은 우선순위로 반영
    });
  }, [value]);

  return state;
}
```

사용법:

```js
function Child({ value }) {
  // value가 바뀌어도 무거운 렌더링은 여유 있을 때 처리
  const deferredValue = useDeferredValue(value);

  return <HeavyComponent value={deferredValue} />;
}
```

### 동시성과 Suspense

이 두 훅이 생긴 또 다른 이유는 `Suspense`와의 연동이다. 데이터를 불러오는 동안 `Suspense`가 fallback을 보여주는데, 동시성 기능이 있어야 그 사이에도 현재 화면이 반응성을 잃지 않는다. Suspense 얘기는 다음에 따로 정리할 것.

---

### 📌 정리

`startTransition = "이건 급하지 않아", useDeferredValue = "이 값은 좀 늦게 반영해도 돼"`

#### 핵심

> React의 동시성은 멀티스레딩이 아니다. JS는 여전히 싱글스레드다. 렌더링 작업을 잘게 쪼개서 브라우저 이벤트 루프에 양보하는 방식으로, 높은 우선순위 작업(사용자 입력 등)이 낮은 우선순위 작업(무거운 리렌더링)보다 먼저 처리되도록 만든 것이다.

#### ⭐️ 앞으로 기억할 것

- 1. **`useTransition` vs `useDeferredValue` 선택 기준**
     상태 업데이트를 내가 직접 제어할 수 있으면 `useTransition`, props로 내려오거나 외부 값이면 `useDeferredValue`.

  ```js
  // 내가 setCount를 직접 호출하는 경우
  startTransition(() => setCount(n + 1));

  // 부모에서 내려온 value를 늦게 반영하고 싶은 경우
  const deferred = useDeferredValue(props.value);
  ```

- 2. **controlled input + startTransition 조합은 금지**
     입력창에서 `onChange` → `startTransition` → `setState`로 연결하면 타이핑이 버벅인다. 사용자 입력은 항상 즉시(블로킹) 처리해야 한다.

- 3. **isPending으로 로딩 UI 만들기**
     `useTransition`의 `isPending`은 "동시 렌더링 중"을 알려주는 플래그다. 스피너나 스켈레톤 대신 기존 UI를 흐리게 처리하는 패턴에 쓰기 좋다.
  ```js
  <div style={{ opacity: isPending ? 0.5 : 1 }}>
    <HeavyList />
  </div>
  ```

📌 결론
▎ 동시성 렌더링의 핵심은 "렌더링을 취소 가능한 작은 단위로 쪼개는 것"이다. 멀티스레드가 아니라 우선순위 스케줄링이다.
▎ `useTransition`과 `useDeferredValue`는 둘 다 "이 업데이트는 낮은 우선순위"라고 표시하는 방법인데, 어디서 상태를 바꾸느냐에 따라 선택이 달라진다.
▎ ⭐️ 실무 적용 포인트: 탭 전환, 필터링, 검색 결과처럼 "무거운 리렌더링이 따라오는 상호작용"에 `useTransition`을 붙이면 체감 반응성이 확 올라간다.
