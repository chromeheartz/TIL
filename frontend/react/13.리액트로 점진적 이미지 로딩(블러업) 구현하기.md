# 리액트로 점진적 이미지 로딩(블러업) 구현하기

> **원문**: https://blog.logrocket.com/progressive-image-loading-react-tutorial/ (LogRocket)
> **번역 출처**: https://velog.io/@eunbinn/progressive-image-loading-react-tutorial

큰 이미지가 늦게 뜨면서 화면이 덜컹거리거나(layout shift) 빈 칸이 보이는 게 늘 거슬렸다. 점진적 이미지 로딩(블러업)은 **저화질 미리보기를 먼저 보여주고 → 원본이 로드되면 부드럽게 교체**하는 기법이다. 미디엄·인스타가 쓰는 그 흐릿하다 선명해지는 효과. 직접 컴포넌트로 만드는 법과 라이브러리로 하는 법 둘 다 정리한다.

### 왜 유용한가

원본이 로드되기 전까지 저화질/미리보기 이미지를 표시한다. 핵심 이점:
- 체감 성능 향상
- layout shift 감소
- 느린 연결에서 UX 개선

두 가지 버전을 쓴다: **작은 placeholder(보통 2KB 이하)** + **원본 고화질**.

## 직접 컴포넌트 만들기

### 1단계 — 기본 ProgressiveImg 골격

일단 placeholder를 그대로 보여주는 형태:

```jsx
const ProgressiveImg = ({ placeholderSrc, src, ...props }) => {
  return (
    <img
      {...{ src: placeholderSrc, ...props }}
      alt={props.alt || ""}
      className="image"
    />
  );
};
export default ProgressiveImg;
```

사용:

```jsx
import ProgressiveImg from "./components/ProgressiveImg";
import image from "./images/large.jpg";
import placeholderSrc from "./images/tiny.jpg";

export default function App() {
  return (
    <ProgressiveImg
      src={image}
      placeholderSrc={placeholderSrc}
      width="700"
      height="465"
    />
  );
}
```

### 2단계 — 썸네일 → 원본으로 교체

`useState`로 현재 보여줄 src를 들고, `useEffect`에서 원본을 미리 로드한 뒤 교체한다.

```jsx
import { useState, useEffect } from "react";

const ProgressiveImg = ({ placeholderSrc, src, ...props }) => {
  const [imgSrc, setImgSrc] = useState(placeholderSrc || src);

  useEffect(() => {
    const img = new Image();   // 메모리에서 원본을 미리 로드
    img.src = src;
    img.onload = () => {
      setImgSrc(src);          // 로드 완료되면 원본으로 교체
    };
  }, [src]);

  return (
    <img
      {...{ src: imgSrc, ...props }}
      alt={props.alt || ""}
      className="image"
    />
  );
};
export default ProgressiveImg;
```

> 포인트: `new Image()`로 백그라운드에서 원본을 받아두고, `onload` 시점에만 `<img>`의 src를 바꾼다. 그래서 깜빡임 없이 자연스럽게 전환된다.

### 3단계 — 블러 전환 효과

지금 보여주는 게 placeholder인지 원본인지에 따라 클래스를 다르게 준다.

```jsx
const customClass =
  placeholderSrc && imgSrc === placeholderSrc ? "loading" : "loaded";
```

```jsx
return (
  <img
    // ...
    className={`image ${customClass}`}
  />
);
```

CSS — placeholder는 흐리게(blur), 원본은 transition으로 선명하게:

```css
.loading {
  filter: blur(10px);
  clip-path: inset(0); /* blur가 경계 밖으로 번지는 것 방지 */
}
.loaded {
  filter: blur(0px);
  transition: filter 0.5s linear;
}
```

이러면 placeholder가 흐릿하게 떠 있다가 원본 로드 완료 시 0.5초에 걸쳐 선명해진다.

## 라이브러리로 하기 — react-progressive-graceful-image

직접 안 만들고 싶으면 라이브러리.

```bash
npm i react-progressive-graceful-image
```

```jsx
import ProgressiveImage from "react-progressive-graceful-image";
import image from "./images/large.jpg";
import placeholderSrc from "./images/tiny.jpg";

export default function App() {
  return (
    <ProgressiveImage src={image} placeholder={placeholderSrc}>
      {(src, loading) => (
        <img
          className={`image${loading ? " loading" : " loaded"}`}
          src={src}
          alt="sea beach"
          width="700"
          height="465"
        />
      )}
    </ProgressiveImage>
  );
}
```

**render props 패턴**을 쓴다 — 자식 함수에 `src`(현재 보여줄 이미지)와 `loading`(로딩 중 여부)을 넘겨준다. 우리가 직접 만든 것과 동작은 같고, `new Image()` + state 로직을 라이브러리가 대신 처리.

---

### 📌 정리
`placeholder를 먼저 + new Image()로 원본 프리로드 + onload에 교체 + blur transition`

#### 핵심
> 점진적 이미지 로딩은 작은 placeholder(<2KB)를 먼저 띄우고, `new Image()`로 원본을 백그라운드 로드한 뒤 `onload`에서 `<img>` src를 원본으로 교체하는 기법이다. `filter: blur()` + `transition`으로 흐릿→선명 전환을 준다. 체감 성능과 layout shift가 개선된다. 직접 만들면 `useState`+`useEffect` 몇 줄, 귀찮으면 `react-progressive-graceful-image`(render props).

#### ⭐️ 앞으로 기억할 것
- 1. 원본은 `new Image()`로 미리 받아둔다
  DOM `<img>`를 바로 원본으로 두지 말고, 메모리에서 프리로드 후 `onload`에 교체 → 깜빡임 방지.
  ```jsx
  const img = new Image();
  img.src = src;
  img.onload = () => setImgSrc(src);
  ```
- 2. placeholder는 진짜 작게
  2KB 이하 초저화질. 어차피 blur 처리되니 화질은 무의미, 용량이 중요.
- 3. blur는 clip-path: inset(0)와 함께
  `filter: blur()`만 쓰면 가장자리가 번진다. `clip-path: inset(0)`로 경계 정리.
  ```css
  .loading { filter: blur(10px); clip-path: inset(0); }
  .loaded  { filter: blur(0); transition: filter .5s linear; }
  ```
- 4. width/height는 꼭 지정
  layout shift 방지의 본질. 크기를 미리 잡아둬야 placeholder→원본 교체 때 안 덜컹댄다.
- 5. 라이브러리는 render props
  `react-progressive-graceful-image`는 `(src, loading) => <img/>` 형태. 로직 직접 안 짜도 됨.

📌 결론
▎ "작은 미리보기 → 프리로드 → 교체 → blur 전환" 4단계가 전부다.
▎ 직접 구현은 `useState`+`useEffect`+`new Image()`, CSS는 blur/transition.
▎ ⭐️ 실무 적용 포인트
  - placeholder는 빌드 시 자동 생성(예: LQIP/blurhash, Next.js `placeholder="blur"`)으로 뽑으면 편하다.
  - 화면 밖 이미지는 여기에 `loading="lazy"`나 IntersectionObserver를 더해 더 아낄 수 있다.
  - Next.js라면 `next/image`가 blur placeholder를 내장 지원 — 직접 구현 전에 먼저 확인.
