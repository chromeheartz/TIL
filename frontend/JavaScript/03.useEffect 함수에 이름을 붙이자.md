# useEffect에 이름을 붙이자

  코드베이스 어딘가에 200줄짜리 컴포넌트에 `useEffect`가 4개 있다고 생각해보자. 하나씩 읽으면서 의존성 배열
  따라가고... 뭘 하는 effect인지 파악하는 데 꼬박 1분이 걸린다. 해결책은 엄청 단순하다. 함수에 이름을 붙이면
   된다.

  원문: [Dan Neciu — Start naming your useEffect functions, you will thank me
  later](https://dev.to/daneciu/start-naming-your-useeffect-functions-you-will-thank-me-later-4i4l)

  ---

  ### 익명 이펙트의 문제

  ```javascript
  // 이게 뭐 하는 effect인지 읽어야만 안다
  useEffect(() => {
    document.title = `${count} items`;
  }, [count]);
  ```

  이름 하나만 붙이면 달라진다.

  ```javascript
  useEffect(
    function updateDocumentTitle() {
      document.title = `${count} items`;
    },
    [count],
  );
  ```

  ### 이름 붙이기의 진짜 가치

  #### 전체 흐름이 한눈에 보인다

  ```javascript
  useEffect(function connectToInventoryWebSocket() { ... }, [warehouseId]);
  useEffect(function fetchInitialStock() { ... }, [warehouseId, locationId, connected]);
  useEffect(function resetStockOnLocationChange() { ... }, [locationId]);
  useEffect(function notifyParentOfStockUpdate() { ... }, [stock, onStockChange]);
  ```

  > 웹소켓 연결 → 초기 재고 fetch → 위치 변경 시 리셋 → 부모에게 알림
  > 의존성 배열 안 봐도 흐름이 읽힌다.

  #### 스택 트레이스가 달라진다

  ```
  // 기존
  at (anonymous) @ InventorySync.tsx:14

  // 이름 붙인 후
  at connectToInventoryWebSocket @ InventorySync.tsx:14
  ```

  프로덕션 에러 디버깅할 때 체감이 크다.

  ### 이름 붙이다 보면 문제가 보인다

  이름을 붙이려는데 `and`나 `also`가 필요하다면, **effect를 쪼개야 한다는 신호**다.

  ```javascript
  // "syncWidthAndApplyTheme" — 이름부터 이상함
  useEffect(() => {
    const handleResize = () => setWidth(window.innerWidth);
    window.addEventListener("resize", handleResize);
    if (user?.preferences?.theme) {
      document.body.className = user.preferences.theme;
    }
    return () => window.removeEventListener("resize", handleResize);
  }, [user?.preferences?.theme]);
  ```

  두 가지 일을 하고 있으니 분리하자:

  ```javascript
  // ✅ 쪼갠 후
  useEffect(function trackWindowWidth() {
    const handleResize = () => setWidth(window.innerWidth);
    window.addEventListener("resize", handleResize);
    return () => window.removeEventListener("resize", handleResize);
  }, []);

  useEffect(
    function applyUserTheme() {
      if (user?.preferences?.theme) {
        document.body.className = user.preferences.theme;
      }
    },
    [user?.preferences?.theme],
  );
  ```

  ### 클린업 함수에도 이름을 붙일 수 있다

  return 안의 함수에도 이름을 붙이면 클린업이 뭘 하는지 바로 보인다.

  ```javascript
  useEffect(
    function pollServerForUpdates() {
      const intervalId = setInterval(() => {
        fetch(`/api/status/${serverId}`)
          .then((res) => res.json())
          .then(setServerStatus);
      }, 5000);

      return function stopPollingServer() {
        clearInterval(intervalId);
      };
    },
    [serverId],
  );
  ```

  처음 보면 신선하게 느껴지는데, 클린업이 setup의 "반대쌍"이라는 게 명확하게 보여서 좋다.

  ### 이름 붙이기 어려우면 effect 위치가 잘못된 것이다

  명확한 이름을 붙이기 어렵다는 것 자체가, 그 effect가 다른 곳에 있어야 한다는 신호다.

  #### 파생 상태는 그냥 변수로

  ```javascript
  // ❌ 굳이 effect 쓸 필요 없음
  useEffect(
    function syncFullName() {
      setFullName(`${firstName} ${lastName}`);
    },
    [firstName, lastName],
  );

  // ✅ 그냥 이렇게
  const fullName = `${firstName} ${lastName}`;
  ```

  #### 이벤트에서 하는 일은 이벤트 핸들러에서

  ```javascript
  // ❌
  useEffect(
    function resetFormOnSubmit() {
      if (submitted) { setName(""); setEmail(""); setSubmitted(false); }
    },
    [submitted],
  );

  // ✅
  function handleSubmit() {
    submitForm({ name, email });
    setName("");
    setEmail("");
  }
  ```

  ### 커스텀 훅으로 빼도 이름은 여전히 필요하다

  커스텀 훅 안에 effect를 넣어도 이름은 붙여야 한다.

  ```javascript
  function useWindowWidth() {
    const [width, setWidth] = useState(
      typeof window !== "undefined" ? window.innerWidth : 0,
    );

    useEffect(function trackWindowWidth() {
      const handleResize = () => setWidth(window.innerWidth);
      window.addEventListener("resize", handleResize);
      return () => window.removeEventListener("resize", handleResize);
    }, []);

    return width;
  }
  ```

  언제 커스텀 훅으로 빼고, 언제 인라인으로 둘지 기준:

  | 상황 | 선택 |
  |------|------|
  | 자체 상태 관리 + 재사용 가능 | 커스텀 훅으로 분리 |
  | 한 곳에서만 쓰이는 effect | 이름 붙이고 인라인 유지 |

  ### 실제로 effect가 줄어든 사례

  처음엔 5개:
  - `initializeMapSDK`
  - `synchronizeZoomLevel`
  - `synchronizeCenterPosition`
  - `handleMarkerInteractions`
  - `cleanupStaleMarkerListeners`

  이름 붙이다 보니:
  - `cleanupStaleMarkerListeners`는 `handleMarkerInteractions`의 클린업 쌍 → 통합
  - `synchronizeZoomLevel`과 `synchronizeCenterPosition`은 항상 같이 실행 → `synchronizeMapViewport`로 통합

  결과: **3개**가 됐다.

  ### 📌 정리
  `useEffect 이름 붙이기 정리`

  #### 핵심
  > 새 라이브러리도, 빌드 플러그인도 없다.
  > 함수에 이름 하나 붙이는 것만으로 가독성, 디버깅, 설계까지 바뀐다.

  #### ⭐️ 앞으로 기억할 것

  - 1. 익명 화살표 대신 기명 함수 표현식

  ```javascript
  // ❌
  useEffect(() => { ... }, [dep]);

  // ✅
  useEffect(function fetchUserProfile() { ... }, [userId]);
  ```

  - 2. 이름에 `and`/`also`가 들어가면 쪼개라

    이름을 붙이려는데 "and"가 필요하다 → effect가 두 가지 일을 하고 있다는 신호. 분리하자.

  - 3. return 함수에도 이름을 붙일 수 있다

  ```javascript
  return function stopPollingServer() {
    clearInterval(intervalId);
  };
  ```

    클린업과 setup이 한 쌍이라는 게 명확해진다.

  - 4. 이름 붙이기 어려우면 effect 자체가 잘못된 위치에 있는 것

    - 파생 상태 → 그냥 변수
    - 이벤트에서 하는 일 → 이벤트 핸들러

  - 5. 스택 트레이스가 달라진다

  ```
  // 전: 어디서 터진 건지 모름
  at (anonymous) @ Component.tsx:14

  // 후: 바로 보임
  at fetchUserProfile @ Component.tsx:14
  ```

  📌 결론

  ▎ 이름을 붙이는 행위 자체가 설계를 드러낸다.
  ▎ 이름이 어색하면 → effect가 너무 많은 일을 하거나 위치가 잘못된 것.
  ▎ ⭐️ 코드 리뷰할 때 익명 effect 보이면 이름부터 붙여달라고 하자.
