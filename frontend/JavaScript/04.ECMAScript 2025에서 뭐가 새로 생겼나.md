# ECMAScript 2025에서 뭐가 새로 생겼나

  새 ECMAScript 스펙이 나올 때마다 "이거 언제 쓰냐"싶은 것들이 반은 넘는데, 2025는 좀 달랐다. Iterator 헬퍼랑 Set 메서드는 솔직히 진작에 있었어야 했고, `Promise.try()`는 보자마자 "아, 이거 필요했던 거네"가 바로
  나왔다.

  ### Import 속성과 JSON 모듈

  `with { type: 'json' }` 구문으로 JSON 파일을 안전하게 임포트할 수 있게 됐다. 기존엔 번들러마다 처리 방식이 달라서 JSON 임포트가 은근 번거로웠는데, 이제 스펙 레벨에서 명시적으로 타입을 선언한다.

  ```js
  // 정적 임포트
  import configData from './config.json' with { type: 'json' };

  // 동적 임포트
  const configData = await import('./config.json', { with: { type: 'json' } });
  ```

  "자바스크립트가 아닌 아티팩트를 가져오기 위한 구문적 기반" — 원문의 표현인데, 앞으로 JSON 말고 CSS나 다른 타입도 확장될 여지가 있다는 뜻이다.

  ---

  ### Iterator 헬퍼 메서드

  이게 제일 반가웠다. 배열 메서드처럼 쓸 수 있는데, **중간 배열을 생성하지 않아서** 메모리 효율이 다르다.

  ```js
  const arr = ['a', '', 'b', '', 'c', '', 'd', '', 'e'];

  arr
    .values()               // 이터레이터 생성
    .filter(x => x.length > 0)
    .drop(1)                // 첫 번째 건너뜀
    .take(3)                // 3개만
    .map(x => `=${x}=`)
    .toArray();
  // ['=b=', '=c=', '=d=']
  ```

  #### 메서드 분류

  | 카테고리 | 메서드 |
  |---------|--------|
  | 이터레이터 반환 | `filter()`, `map()`, `flatMap()` |
  | 불리언 반환 | `some()`, `every()` |
  | 값 반환 | `find()`, `reduce()` |
  | 부작용만 | `forEach()` |
  | 이터레이터 고유 | `drop()`, `take()`, `toArray()` |

  배열과 달리 **점진적으로 계산**된다. 최종적으로 `toArray()`나 `forEach()` 같은 소비 메서드를 붙여야 실행이 시작된다.

  ### Set 메서드 확장

  수학 시간에 배운 집합 연산이 드디어 네이티브로 들어왔다. 직접 구현하거나 lodash 쓰던 거 이제 필요 없다.

  ```js
  const a = new Set(['a', 'b', 'c']);
  const b = new Set(['b', 'c', 'd']);

  a.union(b);                  // Set(['a', 'b', 'c', 'd']) — 합집합
  a.intersection(b);           // Set(['b', 'c'])            — 교집합
  a.difference(b);             // Set(['a'])                 — 차집합
  a.symmetricDifference(b);    // Set(['a', 'd'])            — 대칭차 (XOR)

  a.isSubsetOf(new Set(['a', 'b', 'c', 'd']));   // true — 부분집합
  a.isSupersetOf(new Set(['a', 'b']));            // true — 상위집합
  ```

  기존 Set은 변경되지 않고 **새로운 Set을 반환**한다. 권한 검증(`사용자 권한 ∩ 필요 권한`)같은 데 딱이다.

  ### RegExp.escape()

  사용자 입력값을 동적으로 정규식에 넣을 때, `.` `*` `(` 같은 특수문자 때문에 터지는 경우가 있었는데 이걸 해결해준다.

  ```js
  function removeUnquotedText(str, text) {
    const regExp = new RegExp(
      `(?<!")${RegExp.escape(text)}(?!")`,  // 특수문자 자동 이스케이프
      'gu'
    );
    return str.replaceAll(regExp, '•');
  }

  removeUnquotedText('"yes" and yes and "yes"', 'yes');
  // '"yes" and • and "yes"'
  ```

  단순 포함 여부는 `includes()`로 충분하지만, 동적 정규식이 필요한 검색 기능에서 유용하다.

  ### 정규식 패턴 수정자 (인라인 플래그)

  정규식 전체가 아니라 **특정 부분에만** 플래그를 적용할 수 있다. `(?flag:pattern)` 형식.

  ```js
  /^x(?i:HELLO)x$/.test('xHELLOx');  // ✅ true  — HELLO 부분만 대소문자 무시
  /^x(?i:HELLO)x$/.test('xhellox');  // ✅ true
  /^x(?i:HELLO)x$/.test('XhelloX');  // ❌ false — x 는 여전히 엄격

  /(?s:a.b)/.test('a\nb');            // ✅ true  — . 이 줄바꿈도 매칭
  ```

  ---

  ### 중복된 이름의 캡처 그룹

  동일한 그룹 이름을 `|` 대안의 각 브랜치에 사용할 수 있게 됐다. `v` 플래그 필수.

  ```js
  const RE = /(?<chars>a+)|(?<chars>b+)/v;

  RE.exec('aaa').groups;  // { chars: 'aaa' }
  RE.exec('bb').groups;   // { chars: 'bb' }
  ```

  ❌ 같은 대안 안에서 중복은 여전히 불가. 반드시 다른 `|` 브랜치에만 허용.

  ### Promise.try()

  동기 코드와 비동기 코드를 하나의 Promise 체인으로 묶을 수 있다. 이게 진짜 편하다.

  ```js
  // 기존 방식 — 동기 에러가 체인 밖으로 튀어나올 수 있음
  function computeAsync() {
    const value = syncFuncMightThrow(); // 여기서 throw하면 Promise 체인 바깥
    return asyncFunc(value);
  }

  // Promise.try() 사용
  function computeAsync() {
    return Promise.try(() => {
      const value = syncFuncMightThrow(); // 이제 rejected Promise로 잡힘
      return asyncFunc(value);
    });
  }
  ```

  콜백이 동기로 throw해도 rejected Promise로 감싸줘서 `.catch()`로 처리할 수 있다.

  ### Float16 지원

  16비트 부동소수점. ML 모델 가중치나 GPU 데이터 교환에 쓰는 타입이 네이티브로 들어왔다.

  ```js
  const f16 = Math.f16round(1.337);     // Float16으로 반올림

  const arr = new Float16Array([1.5, 2.5, 3.5]);  // Float16Array

  const view = new DataView(new ArrayBuffer(8));
  view.setFloat16(0, 1.5, true);        // 저장 (리틀엔디안)
  view.getFloat16(0, true);             // 읽기
  ```

  ### 📌 정리
  `ES2025 — 미뤄온 숙제들을 드디어 갚다`

  #### 핵심
  > Iterator 헬퍼, Set 메서드, `Promise.try()`는 이미 많은 팀이 직접 구현하거나 서드파티로 해결하던
  것들이다. 이제 네이티브로 쓸 수 있다는 게 핵심.

  #### ⭐️ 앞으로 기억할 것
  - 1. **Iterator 헬퍼는 lazy하다**
    소비 메서드(`.toArray()`, `.forEach()`)를 붙이기 전까지 실행이 시작되지 않는다. 배열 메서드와 같다고
  생각하면 함정.
    ```js
    const iter = arr.values().filter(x => x > 0).map(x => x * 2);
    // 아직 아무 연산 안 함
    iter.toArray(); // 여기서 실행
    ```

  - 2. **`Promise.try()`는 동기/비동기 경계를 흐린다**
    동기 함수도 Promise 체인 안으로 끌어들일 수 있어서, 에러 처리를 `.catch()` 하나로 통일할 수 있다.
    ```js
    Promise.try(() => JSON.parse(maybeInvalidJson))
      .then(doSomething)
      .catch(handleParseError);
    ```

  - 3. **Set 메서드는 immutable**
    기존 Set을 변경하지 않고 항상 새 Set을 반환한다. 원본 보존 필요할 때 따로 복사 안 해도 된다.

  - 4. **`RegExp.escape()`는 보안 이슈 예방**
    사용자 입력을 정규식 패턴으로 만들 때 이스케이프 빠뜨리면 ReDoS(정규식 서비스 거부 공격) 벡터가 될 수
  있다. 이제 네이티브로 해결.

  📌 결론
  ▎ 2025 스펙은 새로운 개념보다 "이미 쓰고 있던 패턴의 표준화"에 가깝다
  ▎ Iterator 헬퍼 + Set 메서드는 당장 실무에 쓸 수 있는 수준
  ▎ ⭐️ `Promise.try()`는 동기/비동기 경계가 불명확한 유틸 함수에서 바로 써볼 것
  ▎ Float16은 당장은 ML/GPU 쪽이 아니면 쓸 일 없음
