# Blob API

> **출처**: https://velog.io/@sehyunny/intro-to-blob-api-and-its-use-cases (velog @sehyunny, 황세현 번역) / 원문: https://javascript.plainenglish.io/you-dont-know-blob-api-f2c5e9754f29

파일 업로드, 이미지 미리보기, 다운로드 버튼 — 프론트에서 자주 만드는데 그 밑에 깔린 게 다 `Blob`이다. 캔버스 이미지 에디터가 "다운로드" 누르면 바로 받아지는 것도, 큰 파일 청크 업로드도, 로컬 이미지 프리뷰도 전부 Blob API. `Blob` / `File` / `URL.createObjectURL` / `FileReader`의 관계를 한 번에 정리.

### Blob이 뭔데?

> **Blob(Binary Large Object)** = 바이너리 타입의 큰 객체. 보통 이미지·오디오·멀티미디어 파일을 가리킨다.

Blob 객체는 두 프로퍼티를 가진다.

| 프로퍼티 | 의미 |
|---|---|
| `size` | 데이터 크기(바이트) |
| `type` | MIME 타입 |

- Blob은 `blobParts` + 선택적 `type`으로 구성된다.
- **`File` 인터페이스는 `Blob`을 확장**한 것 — 사용자 시스템의 파일을 다루도록 기능을 더했다. (그래서 `<input type="file">`로 받은 File도 Blob 메서드를 다 쓸 수 있다.)

> 핵심 관계: **`File` ⊂ `Blob`**. File은 "이름/수정시간 붙은 Blob"이라고 보면 됨.

### 활용 사례 5가지

#### 1) 청크 업로드 (Chunked Upload)

File이 Blob의 특수 형태라, **`slice` 메서드로 큰 파일을 쪼개서** 나눠 업로드할 수 있다.

![청크 단위 업로드](https://miro.medium.com/proxy/1*EUQYOFD_ZpDeKfKRHNGlSQ.jpeg)

```javascript
const file = new File(["a".repeat(1000000)], "bytefer.txt");
const chunkSize = 40000;
const url = "https://httpbin.org/post";

async function chunkedUpload() {
  for (let start = 0; start < file.size; start += chunkSize) {
    const chunk = file.slice(start, start + chunkSize + 1); // Blob.slice
    const fd = new FormData();
    fd.append("data", chunk);
    await fetch(url, { method: "post", body: fd }).then((res) => res.text());
  }
}
```

> 큰 파일을 한 방에 안 올리고 잘라서 보내니, 실패 시 재시도·진행률 표시·서버 부담 분산에 유리.

#### 2) 인터넷에서 데이터 받아 Blob으로

XHR로 받는 방식:

```javascript
const downloadBlob = (url, callback) => {
  const xhr = new XMLHttpRequest();
  xhr.open('GET', url);
  xhr.responseType = 'blob'; // ← 응답을 Blob으로 받음
  xhr.onload = () => callback(xhr.response);
  xhr.send(null);
};
```

Fetch API로 이미지를 받아 표시하는 방식(더 현대적):

```javascript
const myImage = document.querySelector('img');
fetch(new Request('flowers.jpg'))
  .then((response) => response.blob())     // 응답 → Blob
  .then((myBlob) => {
    const objectURL = URL.createObjectURL(myBlob); // Blob → 임시 URL
    myImage.src = objectURL;
  });
```

#### 3) 이미지 압축 (Canvas 활용)

캔버스에 이미지를 그린 뒤 `toDataURL(type, quality)`로 품질을 낮춰 압축한다.

```javascript
const MAX_WIDTH = 800;

function compress(base64, quality, mimeType) {
  const canvas = document.createElement("canvas");
  const img = document.createElement("img");
  img.crossOrigin = "anonymous";
  return new Promise((resolve) => {
    img.src = base64;
    img.onload = () => {
      // MAX_WIDTH 넘으면 비율 유지하며 축소
      let targetWidth, targetHeight;
      if (img.width > MAX_WIDTH) {
        targetWidth = MAX_WIDTH;
        targetHeight = (img.height * MAX_WIDTH) / img.width;
      } else {
        targetWidth = img.width;
        targetHeight = img.height;
      }
      canvas.width = targetWidth;
      canvas.height = targetHeight;
      const ctx = canvas.getContext("2d");
      ctx.clearRect(0, 0, targetWidth, targetHeight);
      ctx.drawImage(img, 0, 0, canvas.width, canvas.height);
      const imageData = canvas.toDataURL(mimeType, quality / 100); // 품질 0~1
      resolve(imageData);
    };
  });
}
```

Data URL ↔ Blob 변환도 가능하다.

```javascript
// Data URL → Blob
function dataUrlToBlob(base64, mimeType) {
  const bytes = window.atob(base64.split(",")[1]); // base64 디코드
  const ab = new ArrayBuffer(bytes.length);
  const ia = new Uint8Array(ab);
  for (let i = 0; i < bytes.length; i++) ia[i] = bytes.charCodeAt(i);
  return new Blob([ab], { type: mimeType });
}
```

> 사실 캔버스는 `toBlob(callback, mimeType, quality)`도 제공한다 — Data URL 거치지 않고 **바로 Blob**으로 받는 게 더 깔끔하고 메모리 효율적.
> ```javascript
> canvas.toBlob(callback, mimeType, qualityArgument);
> ```

#### 4) Blob URL(Object URL) 만들기

> **Blob URL / Object URL** = Blob·File 객체를 이미지 src나 다운로드 링크로 쓸 수 있게 해주는 **유사 프로토콜(pseudo-protocol)**.

`URL.createObjectURL`이 `blob:<origin>/<uuid>` 형태의 고유 URL을 만든다.

```
blob:https://example.org/40a5fb5a-d56d-4a33-b4e2-0acf6a8e5f641
```

⭐ **메모리 관리 주의**: 다 쓰면 `URL.revokeObjectURL`로 해제해야 한다. 안 그러면 페이지 사는 동안 Blob이 메모리에 남는다.

다운로드 버튼 구현:

```javascript
const download = (fileName, blob) => {
  const link = document.createElement("a");
  link.href = URL.createObjectURL(blob);
  link.download = fileName;
  link.click();
  link.remove();
  URL.revokeObjectURL(link.href); // ⭐ 즉시 해제
};

document.querySelector("#downloadBtn").addEventListener("click", () => {
  const myBlob = new Blob(["You Don't Know Blob API"], { type: "text/plain" });
  download("blob.txt", myBlob);
});
```

#### 5) Blob → Data URL (로컬 미리보기)

> **Data URL** = 바이너리를 텍스트 문자열로 임베드. 형식: `data:[<mediatype>][;base64],<data>`

`FileReader.readAsDataURL`로 업로드 직후 **서버 없이 로컬 미리보기**:

```html
<input type="file" accept="image/*" onchange="loadFile(event)">
<img id="output"/>
<script>
  const loadFile = function (event) {
    const reader = new FileReader();
    reader.onload = function () {
      document.querySelector('#output').src = reader.result; // data URL
    };
    reader.readAsDataURL(event.target.files[0]);
  };
</script>
```

서버(Express)에서 Data URL을 받아 파일로 저장:

```javascript
app.post('/upload', function (req, res) {
  const imgData = req.body.imgData;
  const base64Data = imgData.replace(/^data:image\/\w+;base64,/, ""); // 헤더 제거
  const dataBuffer = Buffer.from(base64Data, 'base64');
  fs.writeFile("image.png", dataBuffer, (err) =>
    err ? res.send(err) : res.send("Image upload successfully!")
  );
});
```

`FileReader`는 용도별 메서드가 더 있다.

| 메서드 | 결과 |
|---|---|
| `readAsDataURL()` | Blob → Data URL (미리보기/임베드) |
| `readAsArrayBuffer()` | Blob → ArrayBuffer (바이너리 가공) |
| `readAsText()` | Blob → 텍스트 |

```javascript
const fileReader = new FileReader();
fileReader.onload = function () {
  const arrayBuffer = fileReader.result;
};
fileReader.readAsArrayBuffer(blob);
```

변환 정리:
- **Blob → ArrayBuffer**: `FileReader.readAsArrayBuffer()`
- **ArrayBuffer → Blob**: `new Blob([new Uint8Array(data)])`

### 📌 정리
`Blob = 바이너리 덩어리. File은 그 확장. createObjectURL로 URL화, FileReader로 Data URL/ArrayBuffer화`

#### 핵심
> `Blob`은 이미지·파일 같은 바이너리 덩어리고 `size`/`type`을 가진다. `File`은 Blob의 확장(이름 붙은 Blob)이라 `<input type="file">`로 받은 파일도 Blob 메서드를 다 쓴다. 핵심 변환은 두 갈래 — ① **Blob → URL**: `URL.createObjectURL`로 `blob:...` 임시 URL을 만들어 `<img src>`나 다운로드 링크로 쓰고, 끝나면 **`revokeObjectURL`로 반드시 해제**. ② **Blob → 문자열/버퍼**: `FileReader`로 `readAsDataURL`(미리보기), `readAsArrayBuffer`(바이너리 가공), `readAsText`. 이걸로 청크 업로드(`file.slice`), fetch `.blob()` 다운로드, 캔버스 `toBlob`/`toDataURL` 이미지 압축, 로컬 미리보기까지 다 풀린다.

#### ⭐️ 앞으로 기억할 것
- 1. **File ⊂ Blob**
  File은 Blob 확장. 그래서 업로드 파일에 `slice`, `size`, `type` 다 쓸 수 있음. 청크 업로드 = `file.slice(start, end)` 반복.
- 2. **`URL.createObjectURL` ↔ `revokeObjectURL` 짝**
  ⭐ createObjectURL 했으면 다 쓰고 **revoke 필수**. 안 하면 메모리 누수(페이지 살아있는 내내 Blob 유지).
- 3. **다운로드 패턴**
  `<a download>` + `href=createObjectURL(blob)` + `.click()` → 프로그램적 파일 다운로드.
- 4. **로컬 미리보기는 `FileReader.readAsDataURL`**
  서버 안 거치고 업로드 즉시 `<img src>`에 꽂음. (대안: `createObjectURL`이 더 가벼움 — base64 인코딩 안 하니까)
- 5. **이미지 압축은 캔버스 `toBlob` 우선**
  `toDataURL`은 base64 문자열이라 무겁고 변환 단계 많음. `canvas.toBlob(cb, type, quality)`로 바로 Blob 받는 게 메모리 효율적.
- 6. **FileReader 메서드별 용도**
  readAsDataURL(미리보기) / readAsArrayBuffer(바이너리 조작) / readAsText(텍스트).

📌 결론
▎ Blob은 프론트에서 바이너리를 다루는 공통 화폐다 — 업로드·다운로드·미리보기·압축이 전부 Blob 변환의 변주.
▎ 두 변환 축만 기억하면 된다: createObjectURL(→URL) / FileReader(→DataURL·ArrayBuffer·Text).
▎ ⭐️ 실무 적용 포인트
▎ - 이미지 미리보기: base64 안 거치는 `URL.createObjectURL(file)`이 `readAsDataURL`보다 가볍다. 단 revoke 잊지 말 것.
▎ - 큰 파일 업로드: `file.slice`로 청크 분할 → 진행률·재시도 구현 + 서버 메모리 부담↓.
▎ - 캔버스 이미지 압축/다운로드: `toDataURL`보다 `toBlob`을 기본으로.
▎ - 다운로드 버튼: createObjectURL + `<a download>` + click + revoke 4단 콤보가 정석.
