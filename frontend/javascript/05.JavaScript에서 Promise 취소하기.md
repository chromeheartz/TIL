# 자바스크립트에서 프로미스 취소하기

> **원문**: https://webdeveloper.beehiiv.com/p/cancel-promises-javascript
> **번역 출처**: https://velog.io/@eunbinn/cancel-promises-javascript

![](https://velog.velcdn.com/images/chromeheartz/post/2fae0fed-8031-4d04-8362-b15638053483/image.png)

JS 프로미스에는 기본 취소 API가 없다. 그런데 검색창 입력이나 버튼 연속 클릭처럼 이전 요청을 무시해야 하는 상황은 실무에서 꽤 자주 나온다. 어떻게 처리하는지 잘 정리된 글을 읽어서 정리해둠.

---

### 핵심 전제

JavaScript의 Promise는 취소할 수 없다. 대신 두 가지 방법으로 **결과를 무시하거나 거부**할 수 있다:

1. `Promise.withResolvers()` — 조기 거부(early rejection)
2. `AbortController` — fetch 요청 자체를 중단

---

### 방법 1: Promise.withResolvers()로 조기 거부

`Promise.withResolvers()`는 promise 객체와 resolve/reject 함수를 한 번에 꺼낼 수 있는 최신 API다.

```javascript
// 기존 방식
let resolve, reject;
const promise = new Promise((res, rej) => {
  resolve = res;
  reject = rej;
});

// 새로운 방식
const { promise, resolve, reject } = Promise.withResolvers();
```

이걸 활용해서 취소 가능한 태스크를 만든다:

```typescript
const buildCancelableTask = <T>(asyncFn: () => Promise<T>) => {
  let rejected = false;
  const { promise, resolve, reject } = Promise.withResolvers<T>();

  return {
    run: () => {
      if (!rejected) {
        asyncFn().then(resolve, reject);
      }
      return promise;
    },
    cancel: () => {
      rejected = true;
      reject(new Error("CanceledError"));
    },
  };
};
```

```typescript
const sleep = (ms: number) => new Promise((res) => setTimeout(res, ms));

const ret = buildCancelableTask(async () => {
  await sleep(1000);
  return "Hello";
});

(async () => {
  try {
    const val = await ret.run();
    console.log("val:", val);
  } catch (err) {
    console.log("err:", err); // 500ms 후 cancel() 호출 시 여기로 떨어짐
  }
})();

setTimeout(() => ret.cancel(), 500);
```

**주의**: 실제로 `asyncFn()` 실행이 멈추는 게 아니다. 백그라운드에서는 계속 돌고 있고, promise만 일찍 거부된다. "취소"라기보다 "결과 무시"에 가깝다.

### 방법 2: AbortController로 fetch 취소

fetch 요청이라면 `AbortController`를 쓰는 게 더 적합하다. signal을 fetch에 전달하면 요청 자체를 중단시킬 수 있다.

```typescript
const buildCancelableTask = <T>(asyncFn: () => Promise<T>) => {
  const abortController = new AbortController();

  return {
    run: () =>
      new Promise<T>((resolve, reject) => {
        const cancelTask = () => reject(new Error("CanceledError"));

        if (abortController.signal.aborted) {
          cancelTask();
          return;
        }

        asyncFn().then(resolve, reject);
        abortController.signal.addEventListener("abort", cancelTask);
      }),
    cancel: () => abortController.abort(),
  };
};
```

`cancel()`을 여러 번 호출해도 `abort` 이벤트는 한 번만 발생한다. Promise.withResolvers 방식과 달리 멱등성이 보장된다.

#### fetch에 signal 직접 전달

```typescript
const buildCancelableFetch = <T>(
  requestFn: (signal: AbortSignal) => Promise<T>,
) => {
  const abortController = new AbortController();

  return {
    run: () =>
      new Promise<T>((resolve, reject) => {
        if (abortController.signal.aborted) {
          reject(new Error("CanceledError"));
          return;
        }
        requestFn(abortController.signal).then(resolve, reject);
      }),
    cancel: () => abortController.abort(),
  };
};

const ret = buildCancelableFetch((signal) =>
  fetch("http://localhost:5000", { signal }).then((res) => res.text()),
);

setTimeout(() => ret.cancel(), 500);
```

**주의**: fetch 취소는 브라우저가 요청을 폐기하는 것이고, 서버 측 처리는 이미 진행 중일 수 있다. GET 요청에는 안전하지만, POST 취소는 서버 상태가 이미 변경됐을 수 있으니 주의.

### React 훅으로 만들기: useSequentialRequest

검색 입력처럼 이전 요청을 취소하고 새 요청을 날려야 하는 패턴을 훅으로 캡슐화한다.

#### 기본 버전

```typescript
import { useCallback, useRef } from "react";

function useLatest<T>(value: T) {
  const ref = useRef(value);
  ref.current = value;
  return ref;
}

export function useSequentialRequest<T>(
  requestFn: (signal: AbortSignal) => Promise<T>,
) {
  const requestFnRef = useLatest(requestFn);
  const currentRequest = useRef<{ cancel: () => void } | null>(null);

  return useCallback(async () => {
    if (currentRequest.current) {
      currentRequest.current.cancel(); // 이전 요청 취소
    }

    const { run, cancel } = buildCancelableFetch(requestFnRef.current);
    currentRequest.current = { cancel };

    return run().finally(() => {
      if (currentRequest.current?.cancel === cancel) {
        currentRequest.current = null;
      }
    });
  }, [requestFnRef]);
}
```

```tsx
export function App() {
  const run = useSequentialRequest((signal) =>
    fetch("http://localhost:5000", { signal }).then((res) => res.text()),
  );
  return <button onClick={run}>Run</button>;
}
```

#### 최적화 버전 (인자 전달 지원)

```typescript
export function useSequentialRequest<Args extends unknown[], Data>(
  requestFn: (signal: AbortSignal, ...args: Args) => Promise<Data>,
) {
  const requestFnRef = useLatest(requestFn);
  const running = useRef(false);
  const abortController = useRef<AbortController | null>(null);

  return useCallback(
    async (...args: Args) => {
      if (running.current) {
        abortController.current?.abort(); // 이전 요청 중단
        abortController.current = null;
      }

      running.current = true;
      const controller = abortController.current ?? new AbortController();
      abortController.current = controller;

      return requestFnRef.current(controller.signal, ...args).finally(() => {
        // race condition 방지: 현재 controller가 맞을 때만 초기화
        if (controller === abortController.current) {
          running.current = false;
        }
      });
    },
    [requestFnRef],
  );
}
```

```tsx
export default function Home() {
  const [data, setData] = useState("");

  const run = useSequentialRequest(async (signal: AbortSignal, query: string) =>
    fetch(`/api/hello?query=${query}`, { signal }).then((res) => res.text()),
  );

  const handleInput = async (queryStr: string) => {
    try {
      const res = await run(queryStr);
      setData(res);
    } catch {
      // 취소된 요청의 에러는 무시
    }
  };

  return (
    <>
      <input
        onChange={(e) => handleInput(e.target.value)}
        placeholder="검색어 입력"
      />
      <div>결과: {data}</div>
    </>
  );
}
```

![최적화 버전 결과](https://media.beehiiv.com/cdn-cgi/image/fit=scale-down,format=auto,onerror=redirect,quality=80/uploads/asset/file/0cf105bd-becf-45ab-9e09-ce88ddfdced9/image.png?t=1720503223)

`finally` 블록에서 `controller === abortController.current`를 비교하는 게 포인트다. 새 요청이 이미 시작됐을 때 이전 요청의 finally가 `running`을 false로 바꿔버리는 race condition을 막는다.

### 📌 정리

`JS Promise는 취소 불가 — 거부하거나 결과를 무시하는 것이다`

#### 핵심

> Promise.withResolvers()는 결과를 조기 거부하고, AbortController는 fetch 요청 자체를 중단시킨다.
> 둘 다 "취소"처럼 보이지만, 백그라운드 비동기 작업은 계속 실행된다는 걸 알아야 한다.

#### ⭐️ 앞으로 기억할 것

- 1. **fetch 취소에는 AbortController + signal**
     signal을 fetch에 직접 넘겨야 브라우저 레벨에서 요청이 중단된다. Promise.withResolvers는 결과만 무시한다.

```typescript
fetch(url, { signal }).then(...)
```

- 2. **useSequentialRequest 패턴**
     새 요청이 오면 이전 요청을 abort하고 새 AbortController를 만든다. 검색 입력, 필터 변경처럼 이전 응답이 필요 없어지는 상황에 딱 맞는 패턴.

- 3. **finally에서 race condition 방지**

```typescript
.finally(() => {
  if (controller === abortController.current) {
    running.current = false;
  }
})
```

현재 실행 중인 controller와 비교해서 이미 다음 요청이 시작됐으면 상태 초기화를 건너뛴다.

- 4. **POST 취소는 조심**
     브라우저가 요청을 폐기해도 서버는 이미 처리 중일 수 있다. GET 요청 취소에는 안전하지만 POST는 서버 상태 변경 여부를 고려해야 한다.

> **📌 결론**
> ▎ JS Promise 자체에 cancel()은 없다 — AbortController나 조기 거부로 우회하는 것이다
> ▎ React에서는 useSequentialRequest 패턴으로 이전 요청 취소를 깔끔하게 캡슐화할 수 있다
> ▎ ⭐️ finally의 controller 비교가 race condition을 막는 핵심이다 — 빠뜨리면 버그난다
