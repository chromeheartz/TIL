# TSRX — JSX의 후계자를 자처하는 TypeScript 언어 확장

> 원문: [TSRX (tsrx.dev)](https://tsrx.dev/) — Dominic Gannaway

**TSRX(TypeScript Render Extensions)** = "에이전트 시대(agentic era)를 위한 선언형 UI를 만드는 TypeScript 언어 확장." 한 줄로는 **"JSX의 spiritual successor(정신적 후계자)"** 를 자처한다. 핵심 아이디어는 **co-location** — 구조(markup)·제어 흐름·스타일·컴포넌트 형태를 **한 파일, 한 블록 안에** 두는 것. 그러면 소스가 그것이 묘사하는 인터페이스에 더 가깝게 읽힌다는 주장.

흥미로운 포인트는 이게 **사람뿐 아니라 AI에게도 유리하다**는 명분이다. "LLM은 관련 정보가 긴 컨텍스트에 흩어져 있을 때보다 가까이 모여 있을 때 더 잘한다"는 연구를 근거로 든다 (앞서 본 [[agentic-engine-optimization]]의 co-location 논리와 같은 결).

### 핵심 특징

- **Framework-agnostic** — React, Preact, Ripple, Solid, Vue로 **컴파일**됨. 전면 재작성 없이 점진적 도입 가능.
- **co-location** — markup/제어흐름/스타일/로직을 한 블록에.
- **TS·JSX 호환 유지** — 완전한 TypeScript 호환.
- **툴링** — Language Server, Prettier·ESLint 플러그인, 에디터 통합(VS Code, Zed, Neovim, IntelliJ, Sublime).
- **상태**: Beta, MIT 라이선스.

### 문법 맛보기

**기본 — `@{ }` 블록** (함수 본문이 곧 렌더. 로직 + markup + scoped `<style>`이 한 블록):

```tsx
export function Greeting({ name }: { name?: string }) @{
  const message = name ? `Hello, ${name}` : 'Hello, stranger';

  <>
    <div class="card">
      <p>{message}</p>
    </div>

    <style>
      .card { padding: 1rem; }
    </style>
  </>
}
```

→ 함수 시그니처 뒤에 `@{ ... }`. JSX처럼 `return` 없이 markup을 직접 쓰고, scoped style까지 같은 블록에.

**제어 흐름 — `@for` / `@if`** (JSX의 `.map()`·삼항 대신 언어 차원 키워드):

```tsx
function UserList({ users, showBio }: Props) @{
  @for (const user of users) {
    const initials = user.name.slice(0, 2).toUpperCase();
    const role = user.admin ? 'Admin' : 'Member';

    <div class="user-row">
      <span class="avatar">{initials}</span>
      <strong>{user.name}</strong>
      <span class="badge">{role}</span>

      @if (showBio && user.bio) {
        const short_bio = user.bio.slice(0, 140);
        const report = () => console.log('viewed ' + user.name);

        <div class="bio-detail">
          <p class="bio">{short_bio}</p>
          <button onClick={report}>Viewed</button>
        </div>
      }
    </div>
  }
}
```

→ **블록 안에서 지역 변수(`initials`, `short_bio`, 핸들러)를 markup 바로 옆에 선언.** `users.map(user => { const initials = ...; return <div>...</div> })` 같은 콜백 보일러플레이트가 사라진다.

**Solid식 lazy prop — `&{ }`** (props 구조분해를 reactive하게 유지):

```tsx
function Counter(&{ count, label }: Props) @{
  <section>
    <h2>{label}</h2>
    <p>Count: {count}</p>
  </section>
}
```

→ Solid에선 props를 그냥 구조분해하면 반응성이 깨지는데, `&{ }`로 lazy destructuring을 문법으로 표현.

**로컬 변수 co-location** (계산 로직을 markup 직전에):

```tsx
function Cart({ items }: { items: Item[] }) @{
  const subtotal = items.reduce((sum, item) => sum + item.price, 0);
  const discount = subtotal > 100 ? 0.1 : 0;

  <div>
    <h2>Your cart</h2>
    <p>Subtotal: ${subtotal}</p>
    <p>Save: ${(subtotal * discount).toFixed(2)}</p>
  </div>
}
```

### 📌 정리

`구조·스타일·제어흐름을 한 블록에 모은, 여러 프레임워크로 컴파일되는 JSX 후계자`

#### 핵심

> **TSRX**는 함수 뒤 `@{ }` 블록 안에 로직·markup·scoped style·제어흐름(`@for`/`@if`)을 **co-location**하는 TypeScript 언어 확장이다. JSX의 `.map()`/삼항 보일러플레이트 대신 언어 차원의 제어 키워드를 쓰고, 지역 변수를 markup 바로 옆에 선언한다. React/Preact/Solid/Vue/Ripple로 **컴파일**되는 framework-agnostic 설계라 점진 도입 가능. co-location이 사람과 LLM 모두의 가독성을 높인다는 게 "agentic era" 명분. 현재 Beta.

#### ⭐️ 앞으로 기억할 것

- 1. **`@{ }` = 렌더 블록**
     `function X(props) @{ ...로직...; <markup/>; <style/> }`. return 없이 markup 직접, scoped style 동봉.
- 2. **`@for` / `@if` = 언어 차원 제어 흐름**
  ```tsx
  @for (const user of users) { const x = ...; <div>{x}</div> }
  @if (cond) { <div/> }
  ```
  JSX의 `arr.map(x => <.../>)` 콜백·삼항 지옥을 대체. 블록 안에 지역 변수 자유롭게.
- 3. **`&{ }` = Solid lazy prop destructuring**
     props 반응성을 깨지 않고 구조분해하는 문법.
- 4. **framework-agnostic = 컴파일 타깃이 여럿**
     React/Preact/Solid/Vue/Ripple. 새 런타임이 아니라 **컴파일러**라는 점이 핵심(점진 도입).
- 5. **co-location의 명분이 "AI 친화"로 확장됨**
     "관련 정보가 가까이 모여 있을 때 LLM이 더 잘한다" → 코드 구조 설계가 에이전트 소비까지 고려하는 시대. [[agentic-engine-optimization]]과 같은 흐름.
- 6. **아직 Beta** — 프로덕션 도입은 신중히. 다만 만든 사람(Dominic Gannaway) 이력 때문에 지켜볼 가치.

📌 결론
▎ TSRX = "JSX 후계자" — 구조/스타일/제어/로직을 `@{ }` 한 블록에 모으는 TS 언어 확장.
▎ 차별점: 언어 차원 `@for`/`@if`/`&{ }` + 여러 프레임워크로 컴파일 + co-location.
▎ "agentic era" 프레이밍 — 가독성 좋은 co-location이 사람뿐 아니라 LLM 컨텍스트에도 유리.
▎ ⭐️ 실무 적용 포인트
▎ → 지금 당장 도입할 건 아님(Beta). 다만 "JSX 보일러플레이트 → 언어 차원 제어 흐름"이라는 방향성은 참고할 만.
▎ → Solid/Vue까지 한 문법으로 컴파일된다는 건, 프레임워크 락인을 줄이려는 팀에 잠재적 카드.
▎ → "코드 co-location이 AI 가독성까지 고려"라는 관점은 우리 컴포넌트 구조 설계에도 적용 가능한 사고방식.
