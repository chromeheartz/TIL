# Axios vs Fetch — 뭘 써야 할까

> **원문**: https://meticulous.ai/blog/fetch-vs-axios/
> **번역 출처**: https://velog.io/@eunbinn/Axios-vs-Fetch

![](https://velog.velcdn.com/images/chromeheartz/post/430afbb1-7e7d-4939-872c-55a939c1ec8f/image.png)

"그냥 fetch 쓰면 되지 axios를 왜 깔아?" vs "axios가 편하던데?" — 늘 애매했던 주제. 둘 다 promise 기반 HTTP 클라이언트인데, 자동 JSON 변환·에러 처리·타임아웃 같은 데서 손맛이 꽤 다르다.

### 둘은 뭐가 다른가 (개요)

- **Fetch API**: 브라우저 내장 `fetch()`. 설치 불필요. Node에선 (구버전 기준) 실험 기능 활성화 필요. *(지금은 Node 18+에 기본 내장)*
- **Axios**: 서드파티 라이브러리. CDN/npm/yarn으로 설치. 브라우저·Node 모두 동작.

둘 다 promise를 반환하는(resolve/reject) HTTP 클라이언트다.

### 설치

```bash
npm install axios
# 또는
yarn add axios
```

```javascript
import axios from "axios";
```

```html
<script src="https://cdn.jsdelivr.net/npm/axios/dist/axios.min.js"></script>
```

## 항목별 비교

### 1. 기본 문법

**Fetch** — URL + 옵션 객체. 옵션 없으면 GET.

```javascript
fetch(url);

fetch(url, {
  method: "GET",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({}),
});
```

**Axios** — 여러 형태 지원.

```javascript
axios(url);                       // 기본 GET
axios.get(url, { /* 옵션 */ });   // 메서드 단축
axios(url, { method: "get", headers: {}, data: {} });
axios({ method: "get", url, headers: {}, data: {} });
```

### 2. JSON 데이터 처리

**Fetch** — `.json()` 호출 + `.then()` 두 번.

```javascript
const url = "https://jsonplaceholder.typicode.com/todos";

fetch(url)
  .then((response) => response.json())
  .then(console.log);
```

**Axios** — `response.data`로 **자동 파싱**된 JSON 제공.

```javascript
axios.get(url).then((response) => console.log(response.data));
```

응답 타입 지정도 가능 (`'json'`(기본), `'arraybuffer'`, `'document'`, `'blob'`, `'text'`, `'stream'`):

```javascript
axios.get(url, { responseType: "json" });
```

### 3. 요청 body 자동 직렬화(stringify)

**Axios** — `data`만 넘기면 자동 stringify + `Content-Type: application/json` 기본 적용.

```javascript
const todo = { title: "A new todo", completed: false };

axios
  .post(url, {
    headers: { "Content-Type": "application/json" },
    data: todo,
  })
  .then(console.log);
```

**Fetch** — `JSON.stringify()` 직접 + 헤더 명시 필수.

```javascript
const todo = { title: "A new todo", completed: false };

fetch(url, {
  method: "post",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify(todo),
})
  .then((response) => response.json())
  .then((data) => console.log(data));
```

### 4. 에러 처리 (가장 큰 차이!)

**Axios** — 2xx 범위 밖 상태코드면 **자동으로 promise reject** → `.catch`로 잡힌다.

```javascript
axios
  .get(url)
  .then((response) => console.log(response.data))
  .catch((err) => console.log(err.message));
```

상세 분기:

```javascript
.catch((err) => {
  if (err.response) {
    // 요청은 갔고 서버가 에러 상태로 응답
    const { status, config } = err.response;
    if (status === 404) console.log(`${config.url} not found`);
    if (status === 500) console.log("Server error");
  } else if (err.request) {
    // 요청은 갔지만 응답 없음
    console.log("Error", err.message);
  } else {
    // 그 외 에러
    console.log("Error", err.message);
  }
});
```

**Fetch** — HTTP 에러(404/500)는 **reject되지 않는다!** 네트워크 실패만 reject. 그래서 `response.ok`를 직접 체크해야 한다.

```javascript
fetch(url)
  .then((response) => {
    if (!response.ok) {
      throw new Error(`This is an HTTP error: The status is ${response.status}`);
    }
    return response.json();
  })
  .then(console.log)
  .catch((err) => console.log(err.message));
```

> ⚠️ 이게 fetch 초보자가 제일 많이 당하는 함정. 404가 나도 `.then`으로 들어온다.

### 5. 타임아웃 / 요청 취소

**Axios** — `timeout` 옵션 한 줄 (기본값 `0` = 무제한).

```javascript
axios
  .get(url, { timeout: 4000 })
  .then((response) => console.log(response.data))
  .catch((err) => console.log(err.message));
```

**Fetch** — `AbortController`로 직접 취소.

```javascript
const controller = new AbortController();
const signal = controller.signal;
setTimeout(() => controller.abort(), 4000);

fetch(url, { signal })
  .then((response) => response.json())
  .then(console.log)
  .catch((err) => console.error(err.message));
```

### 6. 성능 / 브라우저 지원

- 성능은 네이티브 **fetch가 약간 빠르지만** 둘 다 비동기라 체감 차이 미미.
- 모던 브라우저는 둘 다 폭넓게 지원. IE11 같은 구형은 폴리필 필요 (es6-promise, github/fetch).

### 그 외 axios가 더 편한 점 (일반적으로 자주 언급)

- **인터셉터(interceptor)**: 요청/응답을 가로채 토큰 주입·공통 에러 처리 등을 중앙화. fetch엔 없어서 wrapper를 직접 짜야 함.
- **동시 요청 헬퍼**: `axios.all` / `Promise.all`로 병렬 요청 (fetch도 `Promise.all`로 가능).
- **업로드/다운로드 진행률**: axios는 progress 콜백 제공, fetch는 ReadableStream으로 수동 구현.

### 📌 정리
`axios=편의 기능 내장(자동 JSON·에러 reject·타임아웃·인터셉터), fetch=내장·가벼움(직접 처리)`

#### 핵심
> 둘 다 promise 기반 HTTP 클라이언트지만, axios는 **자동 JSON 파싱(`response.data`)**, **2xx 외 자동 reject**, **`timeout` 옵션**, **인터셉터** 등 편의 기능이 기본 내장이다. fetch는 브라우저(및 Node18+) **내장이라 설치 불필요·가볍지만**, JSON `.json()`·HTTP 에러 `response.ok` 체크·`AbortController` 취소를 직접 해야 한다. 선택은 결국 프로젝트 성격과 선호.

#### ⭐️ 앞으로 기억할 것
- 1. fetch는 404/500을 에러로 안 친다
  네트워크 실패만 reject. 반드시 `if (!response.ok) throw ...` 체크.
  ```javascript
  if (!res.ok) throw new Error(`HTTP ${res.status}`);
  ```
- 2. axios는 2xx 밖이면 자동 reject
  `.catch`에서 `err.response`(서버 응답 에러) / `err.request`(무응답)로 분기.
- 3. JSON: axios는 자동, fetch는 수동
  axios `response.data` 바로 사용 / fetch `.then(r => r.json())` 한 번 더.
- 4. body: axios는 자동 stringify, fetch는 `JSON.stringify` + 헤더 직접
- 5. 타임아웃: axios `timeout`, fetch `AbortController`
  ```javascript
  const c = new AbortController();
  setTimeout(() => c.abort(), 4000);
  fetch(url, { signal: c.signal });
  ```
- 6. 인터셉터·진행률 필요하면 axios가 유리
  토큰 자동 주입, 공통 에러 처리 중앙화는 axios 인터셉터가 깔끔.

📌 결론
▎ 가볍게·의존성 없이·간단한 요청 → fetch. 편의 기능·인터셉터·세밀한 에러 처리 → axios.
▎ fetch의 함정 두 개(HTTP 에러 미감지, JSON 수동)만 기억하면 fetch만으로도 충분한 경우가 많다.
▎ ⭐️ 실무 적용 포인트
  - 토큰 갱신/공통 에러 핸들링이 많은 앱 → axios 인터셉터로 중앙화.
  - 번들 크기에 민감하거나 엣지/서버리스 → 내장 fetch (Node18+ 기본).
  - fetch로 가더라도 `ok` 체크 + 타임아웃(AbortController)을 감싼 wrapper 하나는 만들어 두기.
